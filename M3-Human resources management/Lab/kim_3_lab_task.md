# Лабораторная работа. Формирование AI-команды, AI-рабочего пространства и перечня работ AI-проекта

> Работа объединяет практическое задание №2 «Создание рабочего пространства и распределение ролей» и практическое задание №4 «Формирование перечня работ и определение логических связей» в единую лабораторную работу.

## 1. Паспорт работы

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами |
| Тема | Формирование AI-команды, AI-рабочего пространства и перечня работ AI-проекта |
| Вид оценочного средства | Практическое задание / лабораторная работа с элементами командного проекта |
| Вид контроля | Текущий контроль |
| Рекомендуемый формат выполнения | В составе AI-команды (малая группа) |
| Рекомендуемые сервисы | Tilda, Wix, Google Sites, Canva Websites, Figma Prototype — для прототипа; Trello (или аналог), Telegram/WhatsApp — для рабочего пространства |
| Основной артефакт | Работающий прототип AI-продукта + пакет проектных документов |
| Формат сдачи | Работающая ссылка на сайт-прототип + презентация AI-проекта + пакет документов |

## 2. Цель работы

Сформировать у обучающихся практические навыки создания AI-команды, распределения AI-ролей, организации рабочего пространства, а также формирования сводного перечня работ AI-проекта и определения логических связей между ними.

## 3. Задачи

- распределить AI-роли внутри команды и настроить рабочее пространство для реализации AI-проекта;
- выбрать и согласовать тематику AI-проекта;
- сформировать перечень работ AI-проекта на основе иерархической структуры работ (WBS);
- определить логические связи и зависимости между работами разных AI-ролей;
- собрать индивидуальные результаты в единый сводный документ AI-проекта.

## 4. Проверяемый результат обучения

Обучающийся умеет формировать состав AI-команды, распределять зоны ответственности между AI-ролями, организовывать совместную работу, строить перечень работ AI-проекта и согласовывать зависимости между работами разных направлений.

## 5. Тематика AI-проектов

Команда выбирает и согласовывает одну тему для реализации в качестве AI-проекта:

1. Умный AI-ассистент для планирования учебной нагрузки;
2. AI-платформа для коллективного создания конспектов и учебных материалов;
3. AI-сервис подбора и бронирования времени для консультаций с преподавателями;
4. AI-биржа студенческих стажировок и практик от вуза;
5. AI-приложение для формирования студенческих сообществ по интересам;
6. Система адаптивного AI-обучения и подготовки к экзаменам;
7. AI-сервис мониторинга ментального здоровья и антистресс-поддержки;
8. Цифровой AI-портфолио-конструктор для студента;
9. Интерактивный AI-гайд по университетским ресурсам;
10. AI-сервис адаптации и знакомств на основе расписания и интересов.

Допускается предложить собственную тему AI-проекта при условии согласования с преподавателем.

## 6. Ход выполнения

### Часть 1. Формирование AI-команды и рабочего пространства

1. Проведите установочное собрание AI-команды.
2. Распределите AI-роли между участниками (см. раздел 7).
3. Совместно изучите техническое задание на AI-проект.
4. Обсудите и выберите тему AI-проекта из раздела 5 (или предложите собственную).
5. Создайте общий чат AI-команды для коммуникации (Telegram/WhatsApp).
6. Заведите Trello-доску (или аналог) для управления работами AI-проекта, с доступом для преподавателя.

**Результат части 1:** матрица ответственности (RACI), чек-лист понимания требований, ссылки на рабочее пространство.

### Часть 2. Разработка структуры декомпозиции работ верхнего уровня

Ответственный: **AI-менеджер проекта** совместно с AI-лидером команды.

1. Разработайте иерархическую структуру работ (WBS) верхнего уровня AI-проекта, выделив не менее 5 основных фаз.
2. Представьте структуру в двух формах:
   - вложенный список с отображением уровней (1.0, 1.1, 1.2 и т.д.);
   - графическая схема (дерево работ AI-проекта).

### Часть 3. Формирование перечня работ по AI-ролям

Каждый участник AI-команды формирует перечень работ по своему направлению в соответствии с назначенной AI-ролью (см. раздел 7), указывая для каждой работы: код, название, направление, краткое описание, предполагаемого исполнителя и предполагаемый результат (артефакт).

Минимальное число работ по направлениям:

| AI-роль | Минимум работ |
|---|---:|
| AI-аналитик | 8 |
| AI-разработчик | 12 |
| AI-тестировщик | 8 |
| AI-специалист по инфраструктуре | 6 |
| AI-архитектор | 6 |
| AI-дизайнер | 6 |

### Часть 4. Определение логических связей

1. Для каждой работы своего направления определите **внутренние зависимости** — какие работы внутри направления должны быть выполнены раньше.
2. Определите **внешние зависимости** — какие работы других AI-ролей должны предшествовать вашим работам и какие работы зависят от результатов вашего направления.
3. Представьте зависимости в виде списка непосредственных предшественников для каждой работы.

### Часть 5. Формирование сводного перечня работ AI-проекта

Ответственный: **AI-менеджер проекта**.

1. Соберите заполненные перечни работ от всех участников AI-команды.
2. Объедините их в единый сводный перечень с единой сквозной кодификацией.
3. Проведите анализ на дублирование и пропуски работ, при необходимости запросите уточнения у соответствующих AI-ролей.
4. На основе полученных данных составьте матрицу зависимостей между направлениями AI-проекта.
5. Организуйте обсуждение с AI-командой для согласования спорных связей, зафиксируйте решения в протоколе.

### Часть 6. Проверка перечня работ на циклы, тупики и согласованность оценок

Ответственный: **AI-менеджер проекта**.

1. Проверьте сводный перечень работ на наличие циклических зависимостей и тупиков (работ без необходимых последующих связей). При обнаружении проблем скорректируйте связи совместно с AI-командой.
2. Проведите анализ оценок длительности работ, полученных от участников AI-команды, и выявите логические несоответствия между связанными работами (например, разработка модуля оценена короче, чем его тестирование).
3. Согласуйте корректировки с AI-командой.

### Часть 7. Формирование итогового документа и прототипа

1. AI-менеджер проекта формирует итоговый сводный документ AI-проекта (см. раздел 9).
2. AI-команда реализует работающий прототип AI-продукта на выбранном сервисе (Tilda, Wix, Google Sites, Canva Websites, Figma Prototype), демонстрирующий основную концепцию и функционал.
3. AI-команда готовит презентацию AI-проекта.

## 7. AI-роли в команде

### 7.1. AI-лидер команды

Функциональные обязанности: общая координация AI-команды; распределение AI-ролей; согласование итоговой темы AI-проекта; представление результатов на защите.

### 7.2. AI-менеджер проекта

Функциональные обязанности: разработка единых форматов представления информации; настройка рабочего пространства (Trello, чат); разработка WBS верхнего уровня; сбор и интеграция перечней работ от всех AI-ролей; формирование сводного перечня работ и матрицы зависимостей; проверка на циклы, тупики и согласованность оценок; формирование итогового сводного документа.

*Практические задания:*

1. Разработка WBS верхнего уровня (не менее 5 фаз, в виде списка и графической схемы).
2. Разработка единого шаблона таблицы для сбора перечней работ (код, название, направление, описание, исполнитель).
3. Формирование сводного перечня работ с единой сквозной кодификацией.
4. Выявление и согласование межфункциональных зависимостей, оформление протокола.
5. Проверка связей на циклы и тупики.
6. Проверка согласованности оценок длительности.

### 7.3. AI-аналитик

Функциональные обязанности: формирование перечня работ по направлению аналитики и требований AI-продукта; определение зависимостей между аналитическими работами и другими направлениями; оценка длительности аналитических работ.

*Практические задания:*

1. Составить перечень работ по аналитике AI-продукта (не менее 8 работ: от сбора требований до передачи документации команде разработки), с кодом, названием, описанием и результатом для каждой.
2. Определить внутренние зависимости между аналитическими работами и построить их логическую последовательность.
3. Определить внешние зависимости (что должно предшествовать работе аналитика; что зависит от её завершения).
4. Разработать 3–5 пользовательских сценариев (user stories) в формате «Как [роль пользователя], я хочу [действие], чтобы [цель]» с критериями приёмки.
5. Оценить длительность каждой работы в днях с письменным обоснованием.

### 7.4. AI-архитектор

Функциональные обязанности: разработка общей архитектуры AI-продукта; формирование перечня работ по проектированию решения; определение зависимостей архитектурных работ от результатов аналитики и их влияния на разработку.

*Практические задания:*

1. Составить перечень работ по направлению «Архитектура» (не менее 6 работ): выбор стека технологий, проектирование модулей, схема взаимодействия компонентов и т.д.
2. Разработать схему архитектуры AI-продукта с указанием ключевых модулей и связей между ними.
3. Определить зависимости архитектурных работ от результатов аналитики и зависимости разработки от архитектурных решений.
4. Оценить длительность работ по архитектуре с обоснованием.

### 7.5. AI-разработчик

Функциональные обязанности: формирование перечня работ по созданию AI-продукта; определение технологических зависимостей между работами разработки и с другими направлениями; оценка длительности разработки.

*Практические задания:*

1. На основе требований аналитика составить перечень работ по разработке (не менее 12 работ, от настройки проекта до сборки релизной версии), сгруппированных по модулям.
2. Разработать схему технологических зависимостей между модулями AI-продукта.
3. Определить внутренние зависимости работ разработки с учётом последовательности и зависимостей модулей.
4. Определить внешние зависимости: какие работы аналитика/архитектора и инфраструктуры должны быть завершены до начала разработки.
5. Выявить работы, которые могут выполняться параллельно, с учётом ограниченности ресурсов команды.
6. Оценить длительность работ по разработке с учётом параллелизации.

### 7.6. AI-дизайнер

Функциональные обязанности: формирование перечня работ по проектированию интерфейса и пользовательского опыта AI-продукта; определение зависимостей дизайна от аналитики и его влияния на вёрстку прототипа.

*Практические задания:*

1. Составить перечень работ по направлению «Дизайн» (не менее 6 работ): от подготовки макетов до финального UI-кита.
2. Определить, какие результаты аналитики необходимы до начала дизайна и какие работы разработки зависят от готовности макетов.
3. Оценить длительность дизайн-работ с обоснованием.

### 7.7. AI-тестировщик

Функциональные обязанности: формирование перечня работ по тестированию и обеспечению качества AI-продукта; определение зависимостей между тестированием и разработкой; оценка длительности тестирования.

*Практические задания:*

1. Составить перечень работ по тестированию (не менее 8 работ: от планирования до оформления отчётов).
2. Разработать перечень тестовой документации (тест-план, набор тест-кейсов, отчёт о тестировании и т.д.) с указанием сроков подготовки.
3. Для каждой работы тестирования определить, какие работы разработки должны быть завершены до её начала (функциональное, интеграционное, регрессионное тестирование).
4. Описать циклы тестирования и учёт времени на исправление дефектов.
5. Оценить длительность работ по тестированию, включая длительность циклов регрессионного тестирования.
6. Разработать чек-лист приёмочного тестирования (не менее 10 пунктов) с ожидаемым результатом для каждого пункта.

### 7.8. AI-специалист по инфраструктуре

Функциональные обязанности: формирование перечня работ по настройке инфраструктуры и развёртыванию AI-продукта; определение зависимостей инфраструктурных работ от других направлений; оценка длительности инфраструктурных работ.

*Практические задания:*

1. Составить перечень работ по инфраструктуре (не менее 6 работ): настройка сред, репозитория, CI/CD и т.д.
2. Разработать схему инфраструктуры AI-проекта (среды, компоненты, связи между ними).
3. Определить зависимости инфраструктурных работ от решений аналитика/архитектора и влияние результатов на другие направления.
4. Определить работы, блокирующие начало разработки, и работы, которые могут выполняться параллельно с ней.
5. Оценить длительность инфраструктурных работ с обоснованием.
6. Описать меры безопасности AI-продукта (управление доступом, резервное копирование, мониторинг).

## 8. Основной пакет документов (сдаётся в электронном виде)

а) **Матрица ответственности (RACI)** — таблица в Word/Google Docs или слайд в PowerPoint с указанием ФИО, AI-роли, зон ответственности и контактных данных участника.

б) **Чек-лист понимания требований** — проблема и решение, целевая аудитория AI-продукта, вопросы к преподавателю, подпись каждого участника.

в) **Ссылки на рабочее пространство** — Trello-доска с карточками по этапам AI-проекта (доступ для преподавателя), общий чат команды.

г) **Сводный перечень работ AI-проекта** — единая таблица со сквозной кодификацией, содержащая код, название, направление, длительность, предшественников для каждой работы.

д) **Матрица зависимостей между направлениями** и протокол согласования спорных связей.

е) **Прототип AI-продукта** — работающая ссылка на сайт-прототип.

ж) **Презентация AI-проекта**.

## 9. Итоговый результат работы AI-команды

На основе выполненных индивидуальных заданий AI-менеджер проекта формирует единый документ, содержащий:

1. титульный лист с указанием темы AI-проекта, дисциплины и состава AI-команды;
2. сводный перечень работ AI-проекта с указанием кода, названия, направления, длительности в днях, предшественников для каждой работы;
3. описание логических связей — краткое пояснение наиболее важных зависимостей между направлениями;
4. обоснование оценок длительности по каждому направлению;
5. приложения — индивидуальные материалы каждого участника AI-команды по всем выполненным заданиям.

Каждый участник AI-команды сдаёт преподавателю свои индивидуальные материалы. AI-менеджер проекта сдаёт итоговый сводный документ и протоколы координации работы команды.

## 10. Сроки и порядок сдачи

- Срок выполнения устанавливается преподавателем.
- Формат сдачи:
  - ZIP-архив со всеми документами;
  - ссылка на облачную папку (Google Drive/Яндекс.Диск);
  - отдельная ссылка на Trello-доску (с доступом для преподавателя);
  - работающая ссылка на прототип AI-продукта.
- Защита: краткий отчёт AI-лидера команды на занятии (3 минуты) с демонстрацией прототипа и презентации.

## 11. Требования к отчёту

Отчёт AI-команды должен содержать:

1. описание выбранного AI-проекта и его целевой аудитории;
2. состав AI-команды и распределение AI-ролей (RACI);
3. описание рабочего пространства AI-проекта;
4. структуру декомпозиции работ (WBS) верхнего уровня;
5. сводный перечень работ AI-проекта по всем AI-ролям;
6. описание логических зависимостей между работами и направлениями;
7. результаты проверки на циклы, тупики и согласованность оценок;
8. описание прототипа AI-продукта и ссылку на него;
9. вывод о применимости выбранной структуры работ и рисках AI-проекта.

## 13. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- пояснения структуры WBS и формата перечня работ;
- генерации черновых формулировок работ по направлению;
- проверки текста отчёта и пользовательских сценариев;
- поиска логических несоответствий в зависимостях и оценках.

Обязательно указывается:

1. какой инструмент использовался;
2. для какой части работы;
3. какие фрагменты были изменены после проверки;
4. как участник AI-команды убедился в корректности результата.

Не допускается сдача перечня работ, зависимостей или отчёта, полностью сгенерированных ИИ без понимания и самостоятельной проверки логики AI-проекта.

## Пересчёт в БРС

Работа оценивается по единой 10-балльной шкале ФОС. Вес данной работы в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_работы / 10) × вес_работы
```
