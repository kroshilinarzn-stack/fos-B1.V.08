# КИМ_1_Test. Основы проектного управления

## Варианты индивидуализации задания

---

### Вариант A: Устное собеседование по изученным темам

**Формат:** Диалог преподавателя со студентом в формате «вопрос — ответ».

**Вопросы (кейсы) для обсуждения:**

1. Представьте, что вам нужно запустить ИИ-проект в компании с нуля. Опишите последовательность ваших действий от идеи до запуска.

2. Какие факторы, по вашему мнению, являются ключевыми для успеха ИИ-проекта?

3. Какие ошибки чаще всего допускают при управлении ИИ-проектами? Как их избежать?

4. Как бы вы объяснили топ-менеджменту компании необходимость внедрения ИИ и какие аргументы использовали бы?

5. Опишите структуру команды для ИИ-проекта и распределение ответственности между участниками.

6. Представьте, что в середине ИИ-проекта выясняется, что собранных данных недостаточно для достижения целевых показателей точности модели. Ваши действия как менеджера проекта? Какие изменения в документации и планах вам потребуется внести?

7. В компании принято решение внедрить ИИ-решение в ключевой бизнес-процесс, но топ-менеджмент не готов выделить дополнительный бюджет на создание проектного офиса и предпочитает использовать существующую функциональную структуру управления. Какие риски это создает для проекта и как вы, как менеджер проекта, будете их минимизировать?

8. У вас есть два потенциальных ИИ-проекта: первый — разработка чат-бота для внутренней поддержки сотрудников (небольшой бюджет, короткие сроки, низкая неопределенность), второй — создание системы прогнозной аналитики для нового рыночного сегмента (крупный бюджет, длительный срок, высокая неопределенность). Какой подход к управлению (методологию, структуру команды, состав документации) вы выберете для каждого проекта и почему?

9. В ходе выполнения ИИ-проекта заказчик просит добавить новую функцию, которая не была предусмотрена в Описании содержания проекта, но, по его мнению, критически важна для бизнеса. При этом сроки и бюджет проекта строго фиксированы. Как вы будете действовать? Какие документы и процессы управления проектом вы задействуете?

10. Проект по созданию ИИ-системы завершен, продукт передан в эксплуатацию. Опишите, из каких этапов состоит процедура закрытия проекта. Какие документы должны быть подготовлены и переданы заказчику?

### Вариант B: Разработка чек-листа для запуска ИИ-проекта

**Формат:** Инфографика или структурированный документ (чек-лист) с комментариями.

**Описание:** Разработайте практический чек-лист для менеджера проекта, который запускает ИИ-проект в компании. Чек-лист должен включать:

1. Этапы запуска проекта (от инициации до планирования) с указанием ключевых действий (LC-1.1).

2. Перечень документов, которые необходимо подготовить на каждом этапе, с кратким описанием их назначения (LC-1.3).

3. Рекомендации по управлению рисками, специфичными для ИИ-проектов (зависимость от данных, неопределенность моделей и т.п.).

---

#### Критерии оценивания

| Критерий | Макс. балл |
|----------|------------|
| Полнота охвата этапов жизненного цикла проекта | 3 |
| Корректность и практичность рекомендаций по документации | 3 |
| Учет специфики ИИ-проектов (данные, неопределенность, MLOps) | 3 |
| Визуальное оформление и удобство использования чек-листа | 1 |

### Вариант C. Сравнительный анализ проектной документации для двух ИИ-проектов

**Формат:** Сравнительная таблица + аналитическая записка (4–6 страниц).

**Описание:** Выберите два реальных или гипотетических ИИ-проекта (например, «Разработка чат-бота для поддержки клиентов» и «Создание системы рекомендаций для интернет-магазина»). Проведите сравнительный анализ:

1. Бизнес-целей и стратегий внедрения ИИ для каждого проекта (LC-1.1).

2. Состава и содержания проектной документации, которая потребуется для каждого проекта (LC-1.3).

3. Отличий в подходах к управлению проектами (например, выбор методологии, управление рисками, работа с данными).

---

#### Критерии оценивания

| Критерий | Макс. балл |
|----------|------------|
| Глубина анализа бизнес-целей и стратегий | 3 |
| Полнота и корректность сравнительного анализа документации | 3 |
| Выявление специфических различий в управлении ИИ-проектами | 3 |
| Структурированность и качество оформления отчета | 1 |

### Вариант D. Инфографика: «Жизненный цикл ИИ-проекта»

**Формат:** Инфографика (постер, схема) + краткое описание (1–2 страницы).

**Описание:** Визуализируйте жизненный цикл ИИ-проекта, включив:

1. Основные фазы (от инициации до завершения) с указанием ключевых этапов (LC-1.1).

2. Перечень проектных документов, которые создаются на каждой фазе (LC-1.3).

3. Особенности управления ИИ-проектами (работа с данными, обучение моделей, MLOps, мониторинг).

| Отражение специфики ИИ-проектов | 2 |
| Качество визуального оформления и удобство восприятия | 2 |
