# КИМ 6. Лабораторная работа "Управление качеством проекта"
# Методические указания для преподавателя

## Назначение

Этот КИМ используется как образец оформления рубежного/текущего контроля в ФОС для модуля, завершающего блок дисциплины. Комбинирует письменный артефакт (план управления качеством) и устную защиту (коллоквиум).

## Рекомендации по проведению

1. До занятия рекомендуется напомнить студентам базовые методы управления качеством (стандартизация, диаграмма Парето) на простом примере вне ИИ-контекста.
2. На занятии целесообразно разобрать построение диаграммы Парето и её интерпретацию.
3. Разработку метрик качества ML-модели и план управления качеством рекомендуется выполнять самостоятельно, с последующей защитой на коллоквиуме.

## Что проверять в первую очередь

- построена ли диаграмма Парето на содержательных (не случайных) данных о дефектах;
- обоснован ли выбор метрик качества модели и их пороговых значений;
- содержит ли план управления качеством распределение ролей и ответственности;
- описан ли мониторинг качества после внедрения (дрейф данных/модели);
- способен ли обучающийся на коллоквиуме самостоятельно объяснить и защитить предложенные решения.

## Типичные ошибки студентов

1. Строят диаграмму Парето на произвольных, не обоснованных данных.
2. Выбирают метрики качества модели без учёта специфики задачи (например, только accuracy для несбалансированных классов).
3. Не связывают план управления качеством с распределением ролей в команде.
4. Не упоминают мониторинг дрейфа данных/модели после внедрения.


## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. диаграммы Парето по 5–7 типовым дефектам;
2. таблицы из 3–4 метрик качества модели с пороговыми значениями;
3. краткого плана управления качеством (цели, процессы, роли);
4. короткой устной защиты (3–5 минут).

## Расширенная версия задания

Для сильной группы можно добавить:

1. сравнение функционально-стоимостного и функционально-физического анализа для кейса;
2. проектирование системы мониторинга дрейфа данных/модели;
3. анализ влияния изменения порогов метрик на бизнес-показатели продукта;
4. сравнительный анализ практик управления качеством нескольких технологических компаний.
