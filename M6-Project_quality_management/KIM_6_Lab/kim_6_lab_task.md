# КИМ 6. Лабораторная работа "Управление качеством проекта"

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Модуль 6. Управление качеством проекта |
| Вид оценочного средства | Лабораторная работа (план управления качеством) + коллоквиум |
| Вид контроля | Рубежный контроль; текущий контроль |
| Рекомендуемый формат выполнения | Индивидуально или в малой команде |
| Рекомендуемая среда | MS Excel / Google Sheets, текстовый редактор |
| Основной артефакт | План управления качеством AI-продукта + диаграмма Парето + набор метрик качества ML-модели |
| Рекомендуемая трудоёмкость | 5 академических часов (лекции, самостоятельная работа, лабораторная работа) + подготовка к коллоквиуму |

## 2. Назначение КИМ

КИМ предназначен для проверки того, что обучающийся умеет применять методы управления качеством в проектах (стандартизацию, функционально-стоимостной и функционально-физический анализ, диаграммы Парето), ориентируется в практике систем менеджмента качества технологических компаний, а также умеет разрабатывать план управления качеством AI-продукта и определять метрики качества ML-моделей.

Так как контроль по модулю является рубежным, задание также проверяет способность обучающегося устно защитить и обосновать предложенные решения в формате коллоквиума.

## 3. Проверяемый результат обучения

Обучающийся подбирает методологию разработки системы ИИ; организует процесс контроля качества AI-продукта, применяет инструменты приоритизации дефектов и метрики качества ML-моделей.

## 4. Проверяемые результаты и компетенции

| Компетенция / область результата | Ориентировочный индикатор | Уровень | Как проявляется в задании |
|---|---|---|---|
| LC-4 | 4.1 | Средний | Разработка плана управления качеством AI-продукта, применение диаграмм Парето для приоритизации дефектов, определение метрик качества ML-моделей. |

## 5. Исходные данные (кейс проекта)

Для выполнения задания используется учебный кейс AI-проекта (может совпадать с кейсом из предыдущих модулей), содержащий:

1. краткое описание AI-продукта и требований к его качеству со стороны заказчика/пользователей;
2. перечень типовых дефектов/проблем качества, выявленных на этапе тестирования или пилотной эксплуатации (может быть предоставлен преподавателем или обоснованно смоделирован обучающимся);
3. сведения о применяемой методологии разработки (например, CRISP-DM/CRISP-ML(Q)).

## 6. Программные средства

Рекомендуемый стек:

- MS Excel / Google Sheets — для построения диаграммы Парето;
- при необходимости — Python (`pandas`, `matplotlib`) для автоматизации расчётов и визуализации;
- текстовый редактор — для оформления плана управления качеством.

## 7. Ход выполнения

### Часть 1. Методы управления качеством в проекте

1. Опишите, какие методы управления качеством применимы к вашему AI-проекту (стандартизация процессов, функционально-стоимостной анализ, функционально-физический анализ и т.д.).
2. Определите, какие требования к качеству предъявляются к продукту со стороны заказчика/пользователей и регуляторных норм (если применимо).

### Часть 2. Стандарты и практика управления качеством

1. Кратко опишите, какие подходы систем менеджмента качества (например, элементы ISO 9001 или отраслевых практик технологических компаний) применимы к разработке AI-продукта.
2. Опишите, как эти подходы адаптируются к специфике разработки ИИ-систем (итеративность, зависимость качества от данных и модели).

### Часть 3. Диаграмма Парето

1. Соберите (или используйте предоставленные) данные о дефектах/проблемах качества AI-продукта с указанием частоты возникновения.
2. Постройте диаграмму Парето для приоритизации задач по устранению дефектов.
3. Сформулируйте, какие 20% причин дают 80% проблем (принцип Парето) и как это влияет на приоритизацию работ.

### Часть 4. Метрики качества ML-модели

1. Определите набор метрик качества, релевантных для вашей задачи (например, accuracy/precision/recall/F1 для классификации, RMSE/MAE для регрессии, специфические бизнес-метрики).
2. Обоснуйте выбор пороговых значений метрик, приемлемых для запуска AI-продукта в эксплуатацию.
3. Опишите, как будет организован мониторинг качества модели после внедрения (например, отслеживание дрейфа данных/модели).

### Часть 5. План управления качеством AI-продукта

Сформируйте план управления качеством, включающий:

1. цели и критерии качества продукта;
2. процессы контроля качества на каждом этапе разработки (данные, модель, интеграция, эксплуатация);
3. роли и ответственность за качество (может быть связано с RACI-матрицей из модуля 3);
4. перечень метрик и порядок их мониторинга;
5. процедуру приоритизации и устранения дефектов на основе диаграммы Парето.

### Часть 6. Подготовка к коллоквиуму

Подготовьте краткую устную защиту (5–7 минут) плана управления качеством, включающую:

1. обоснование выбранных методов и метрик;
2. интерпретацию диаграммы Парето;
3. ответы на вопросы о рисках и ограничениях предложенного плана.

### Часть 7. Дополнительное исследование

Выполните минимум одно дополнительное исследование:

- сравните применимость функционально-стоимостного и функционально-физического анализа для вашего кейса;
- предложите способ мониторинга дрейфа данных/модели и его встраивания в план качества;
- проанализируйте, как изменение порогового значения метрики качества влияет на бизнес-показатели продукта;
- сравните практики управления качеством в 2–3 технологических компаниях (на основе открытых источников) и адаптируйте лучшие практики к своему кейсу.

## 8. Формат сдачи

Обучающийся сдаёт:

1. план управления качеством AI-продукта (документ);
2. диаграмму Парето с интерпретацией;
3. таблицу метрик качества ML-модели с обоснованными пороговыми значениями;
4. (при выполнении) материалы дополнительного исследования;
5. устно защищает план в формате коллоквиума.

## 9. Требования к отчёту

Отчёт должен содержать:

1. описание кейса AI-продукта и требований к качеству;
2. описание применённых методов управления качеством;
3. диаграмму Парето с приоритизацией дефектов;
4. таблицу метрик качества ML-модели с пороговыми значениями;
5. план управления качеством (цели, процессы, роли, мониторинг);
6. вывод о ключевых рисках качества AI-продукта и способах их снижения.

## 10. Контрольные вопросы

1. Какие методы управления качеством применяются в классических проектах и как они адаптируются для AI-проектов?
2. В чём разница между функционально-стоимостным и функционально-физическим анализом?
3. Что такое принцип Парето и как диаграмма Парето помогает приоритизировать дефекты?
4. Какие метрики качества характерны для задач классификации, а какие — для задач регрессии?
5. Почему выбор порогового значения метрики качества модели является управленческим, а не только техническим решением?
6. Что такое дрейф данных и дрейф модели и почему их мониторинг важен для управления качеством AI-продукта?
7. Какие элементы систем менеджмента качества (например, ISO 9001) применимы к разработке ИИ-систем?
8. Как план управления качеством связан с распределением ролей и ответственности в команде (RACI)?
9. Какие риски возникают, если качество AI-продукта оценивается только по техническим метрикам модели, без учёта требований заказчика?
10. Как итеративность разработки ИИ-моделей влияет на процесс управления качеством по сравнению с классическими программными проектами?

## 11. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- пояснения методов управления качеством и стандартов;
- генерации черновой структуры плана управления качеством;
- проверки текста отчёта.

Обязательно указывается:

1. какой инструмент использовался;
2. для какой части работы;
3. какие фрагменты были изменены после проверки;
4. как обучающийся убедился в применимости предложенных решений к своему кейсу.

Не допускается сдача плана управления качеством, полностью сгенерированного ИИ без адаптации к конкретному кейсу; на коллоквиуме обучающийся должен уметь самостоятельно обосновать все ключевые решения.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

Итоговый вклад КИМ в БРС рассчитывается по формуле:

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```

Так как контроль по модулю является рубежным, дополнительно допускается собственный алгоритм формирования рубежной оценки (например, среднее по нескольким КИМ блока), который фиксируется в `docs/evaluation_policy.md`.
