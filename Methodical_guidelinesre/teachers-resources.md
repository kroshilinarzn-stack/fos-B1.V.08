# Методические указания по составлению контрольно-измерительных материалов (КИМ) с использованием учебно-методических ресурсов

## Назначение документа

Настоящие методические указания описывают способы использования учебно-методических ресурсов репозитория для создания и генерации контрольно-измерительных материалов по дисциплине «Управление проектами в сфере ИИ». Материал ориентирован на преподавателей, разрабатывающих новые варианты заданий, тестов, практических и лабораторных работ.

---

## Способы использования ресурсов

### 1. Ручное использование ресурсов

Ручной способ предполагает прямое использование готовых шаблонов, критериев и методических указаний для разработки КИМ «вручную», без применения автоматизированных инструментов.

#### 1.1. Адаптация существующих КИМ

**Порядок действий:**

1. Выберите КИМ из репозитория, наиболее близкий по тематике и типу задания.
2. Изучите комплект документов:
   - Подробное задание — поймите структуру и требования.
   - Шаблон для студента — оцените форму представления.
   - Варианты индивидуализации — изучите возможные модификации.
3. Модифицируйте содержание:
   - Смените кейс/предметную область.
   - Измените исходные данные.
   - Добавьте или исключите отдельные разделы в зависимости от уровня подготовки студентов.
   - Адаптируйте шкалы оценивания под свой курс.
4. Составьте новый комплект документов:
   - Новое задание (на основе подробного задания).
   - Новый шаблон для студента (при необходимости).
   - Обновлённые критерии оценивания.

**При ручной адаптации рекомендуется:**
- Сохранять общую структуру КИМ (тип, количество разделов, логику оценивания).
- Использовать профессиональную терминологию.
- Проверять соответствие таксономии Блума заявленному уровню сложности.
- Согласовывать изменения с рабочей программой дисциплины.

#### 1.2. Разработка новых вариантов заданий

1. На основе вариантов индивидуализации создайте новый набор вариантов.
2. Укажите для каждого варианта:
   - Тематику (отрасль, тип ИИ-решения).
   - Уровень сложности (базовый, средний, продвинутый).
   - Дополнительные условия (этический аспект, объяснимость модели, регуляторные требования).
3. Зафиксируйте правила распределения вариантов между студентами.

### 2. Автоматизированное использование ресурсов

Автоматизированный способ предполагает использование программных средств для генерации КИМ на основе структурированных данных и шаблонов из репозитория.

#### 2.1. Структура данных для автоматизации

Для автоматизированной генерации КИМ используется структурированное описание заданий в формате JSON или YAML. Пример структуры для тестового задания:

```json
{
  "kim_id": "KIM_1_Test",
  "module": 1,
  "title": "Основы проектного управления",
  "type": "test",
  "questions": [
    {
      "id": "q1",
      "type": "single_choice",
      "question": "Что является основным ограничением проекта?",
      "options": ["Сроки", "Бюджет", "Содержание", "Все перечисленные"],
      "correct_answer": 3
    }
  ],
  "variants": [
    {
      "id": "v1",
      "difficulty": "basic",
      "questions_count": 10
    }
  ]
}
```

#### 2.2. Скрипт для генерации КИМ

Ниже приведён пример скрипта на Python для генерации тестовых заданий из структурированных данных.

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

"""
Генератор контрольно-измерительных материалов
Использование: python generate_kim.py --kim KIM_1_Test --variant basic
"""

import json
import argparse
import random
from datetime import datetime
from pathlib import Path


class KIMGenerator:
    """Генератор КИМ на основе JSON-описания"""

    def __init__(self, kim_file: str):
        """Загрузка описания КИМ из JSON-файла"""
        with open(kim_file, 'r', encoding='utf-8') as f:
            self.kim_data = json.load(f)

    def generate_test(self, variant_id: str) -> dict:
        """Генерация тестового задания"""
        variant = self._get_variant(variant_id)
        questions = self._select_questions(variant.get('questions_count', 10))

        return {
            "kim_id": self.kim_data['kim_id'],
            "title": self.kim_data['title'],
            "module": self.kim_data['module'],
            "variant": variant_id,
            "difficulty": variant.get('difficulty', 'standard'),
            "generated_at": datetime.now().isoformat(),
            "questions": questions
        }

    def _get_variant(self, variant_id: str) -> dict:
        """Получение варианта по ID"""
        for v in self.kim_data.get('variants', []):
            if v['id'] == variant_id:
                return v
        return {'id': variant_id, 'difficulty': 'standard', 'questions_count': 10}

    def _select_questions(self, count: int) -> list:
        """Выборка вопросов с учётом сложности"""
        all_questions = self.kim_data.get('questions', [])
        if len(all_questions) <= count:
            return all_questions
        return random.sample(all_questions, count)

    def export_to_markdown(self, test_data: dict) -> str:
        """Экспорт теста в формат Markdown"""
        lines = []
        lines.append(f"# {test_data['title']}")
        lines.append("")
        lines.append(f"**Вариант:** {test_data['variant']}")
        lines.append(f"**Сложность:** {test_data['difficulty']}")
        lines.append(f"**Дата генерации:** {test_data['generated_at']}")
        lines.append("")
        lines.append("---")
        lines.append("")

        for i, q in enumerate(test_data['questions'], 1):
            lines.append(f"## Вопрос {i}")
            lines.append("")
            lines.append(q['question'])
            lines.append("")
            if q['type'] == 'single_choice':
                for j, opt in enumerate(q.get('options', []), 1):
                    lines.append(f"{j}. {opt}")
            lines.append("")

        return "\n".join(lines)

    def export_to_lms_format(self, test_data: dict) -> dict:
        """Экспорт в формат для LMS Moodle (GIFT)"""
        gift_lines = []
        for q in test_data['questions']:
            if q['type'] == 'single_choice':
                question = q['question']
                options = q.get('options', [])
                correct = q.get('correct_answer', 0)
                gift_lines.append(f"::{question}:: {question} {{")
                for i, opt in enumerate(options):
                    if i == correct:
                        gift_lines.append(f"    ={opt}")
                    else:
                        gift_lines.append(f"    ~{opt}")
                gift_lines.append("}")
                gift_lines.append("")
        return {"gift_format": "\n".join(gift_lines)}


def main():
    parser = argparse.ArgumentParser(description='Генерация КИМ из JSON-описания')
    parser.add_argument('--kim', required=True, help='ID КИМ (например, KIM_1_Test)')
    parser.add_argument('--variant', default='basic', help='ID варианта')
    parser.add_argument('--output', '-o', help='Путь для сохранения результата')
    parser.add_argument('--format', choices=['md', 'gift'], default='md', help='Формат вывода')

    args = parser.parse_args()

    kim_file = Path(f"{args.kim}.json")
    if not kim_file.exists():
        print(f"Ошибка: файл {kim_file} не найден")
        return

    generator = KIMGenerator(kim_file)
    test_data = generator.generate_test(args.variant)

    if args.format == 'md':
        output = generator.export_to_markdown(test_data)
        ext = 'md'
    else:
        output = generator.export_to_lms_format(test_data)['gift_format']
        ext = 'txt'

    if args.output:
        output_file = args.output
    else:
        output_file = f"{args.kim}_{args.variant}.{ext}"

    with open(output_file, 'w', encoding='utf-8') as f:
        f.write(output)

    print(f"✅ КИМ сгенерирован: {output_file}")


if __name__ == "__main__":
    main()
```

#### 2.3. Использование скрипта

```bash
# Генерация теста в формате Markdown
python generate_kim.py --kim KIM_1_Test --variant basic --format md

# Генерация теста в формате GIFT для импорта в Moodle
python generate_kim.py --kim KIM_1_Test --variant advanced --format gift

# Генерация с указанием выходного файла
python generate_kim.py --kim KIM_1_Test --variant basic -o test_v1.md
```

### 3. Использование ИИ для генерации КИМ

Применение генеративных ИИ-моделей позволяет быстро создавать новые варианты заданий, адаптировать их под разные контексты и уровни сложности.

#### 3.1. Общие принципы

1. Используйте материалы репозитория как **контекст** для ИИ-модели.
2. Чётко формулируйте требования к результату.
3. Проверяйте сгенерированный материал на корректность.
4. Адаптируйте результат под конкретный кейс.

#### 3.2. Промпты для генерации тестовых заданий

**Промпт 1: Генерация тестовых вопросов (закрытого типа)**

```text
Ты — эксперт в области управления проектами в сфере искусственного интеллекта.
На основе материалов модуля 1 «Проекты и среда управления» составь 10 тестовых вопросов закрытого типа с 4 вариантами ответов (один правильный).

Требования:
1. Вопросы должны охватывать темы: определение проекта, классификация проектов, модель «Железного треугольника», отличия ИИ-проектов от классических.
2. Вопросы должны иметь различную сложность (4 лёгких, 4 средних, 2 сложных).
3. Каждый вопрос должен проверять понимание, а не просто запоминание терминов.
4. Для каждого вопроса укажи правильный ответ.

Формат ответа:
Вопрос 1. [Текст вопроса]
Варианты:
A. [Вариант A]
B. [Вариант B]
C. [Вариант C]
D. [Вариант D]
Правильный ответ: [Буква]
```
**Промпт 2: Генерация вопросов на соответствие**

```text
Составь 5 тестовых заданий на сопоставление по теме «Документация в классических и ИИ-проектах».

Для каждого задания:
1. Приведи список из 4-5 терминов (классические и ИИ-документы).
2. Приведи список соответствующих определений или назначений (перемешанный порядок).
3. Укажи правильные пары соответствия.

Термины для включения: Устав проекта, План управления проектом, Dataset Card, Model Card, MLOps Pipeline, Этический чек-лист.
```
**Промпт 3: Генерация открытых вопросов для коллоквиума**

```text
Составь перечень вопросов для устного коллоквиума по модулю 4 «Управление сроками проекта».

Структура перечня:
1. 5 теоретических вопросов, требующих развёрнутого ответа.
2. 3 практических вопроса с необходимостью выполнения расчётов.
3. Критерии оценки каждого вопроса.

Уровень сложности — продвинутый. Вопросы должны проверять не только знание формул, но и понимание их применения в ИИ-проектах.
```
### 3.3. Промпты для генерации лабораторных и практических работ

**Промпт 4: Генерация варианта лабораторной работы**

```text
Ты — преподаватель дисциплины «Управление проектами в сфере ИИ». Разработай вариант лабораторной работы по модулю 2 «Управление содержанием проекта» для слабой группы студентов.

Используй шаблон из репозитория `kim_2_lab_template.md` как основу.

В сгенерированном задании должны быть:
1. Конкретный кейс из области [указать отрасль, например, здравоохранение].
2. Описание проблемы, которую решает ИИ-система.
3. Задание на разработку Устава проекта, Impact Mapping, User Story Map и WBS с упрощёнными требованиями.
4. Все разделы шаблона должны быть адаптированы под выбранный кейс, а не скопированы из шаблона без изменений.
```
**Промпт 5: Генерация исходных данных для расчётов**

```text
Сгенерируй исходные данные для лабораторной работы по сетевому планированию (модуль 4).

Создай таблицу из 15-20 работ со следующими колонками:
- ID работы (например, A, B, C...)
- Наименование работы (из области ИИ-проекта)
- Длительность (в рабочих днях, от 2 до 20 дней)
- Предшественники (список ID работ, от которых зависит выполнение)

Данные должны быть логически связаны — должна существовать возможность построения связного сетевого графика без циклов.
```
### 3.5. Промпты для адаптации под конкретную группу

**Промпт 6: Адаптация сложности задания**

```text
У меня есть следующее задание по лабораторной работе:

[вставьте текст задания]

Адаптируй это задание для:
1. Слабой группы студентов (упрости требования, уменьши количество артефактов, добавь больше пояснений).
2. Сильной группы студентов (добавь дополнительные исследовательские задачи, усложни кейс, добавь требования к обоснованию решений).

В каждом из двух вариантов сохрани общую структуру задания, но измени содержание и глубину проработки.
```
## Сводная таблица методов использования

| Способ | Когда применять | Преимущества | Ограничения |
|---|---|---|---|
| **Ручное использование** | Разработка новых КИМ, тонкая настройка под конкретную группу | Полный контроль качества, возможность учёта специфики | Трудоёмкость, требует времени |
| **Автоматизированное** | Массовая генерация вариантов, быстрая проверка знаний | Скорость, масштабируемость, воспроизводимость | Ограниченность форматов, требуется подготовка данных |
| **ИИ-генерация** | Быстрая генерация идей, адаптация под разные контексты | Гибкость, скорость, разнообразие вариантов | Требуется проверка корректности, риск галлюцинаций |

## Рекомендации по выбору способа

| Задача | Рекомендуемый способ |
|---|---|
| Разработка нового теста с нуля | Ручное использование + ИИ-генерация идей |
| Генерация 30+ вариантов тестов | Автоматизированное использование |
| Адаптация задания под новую отрасль | ИИ-генерация с последующей ручной проверкой |
| Создание нового шаблона для студента | Ручное использование по образцу |
| Формирование экзаменационных билетов | ИИ-генерация + ручная валидация |
| Быстрое изменение сложности задания | ИИ-генерация с указанием целевого уровня |

## Требования к качеству сгенерированных материалов

Независимо от способа генерации, итоговый КИМ должен соответствовать следующим критериям:

1. **Корректность** — все фактические данные, термины и расчёты должны быть верными.
2. **Однозначность** — формулировки должны исключать двусмысленное толкование.
3. **Соответствие программе** — содержание должно отражать темы модуля.
4. **Проверяемость** — для каждого задания должны существовать объективные критерии оценки.
5. **Адаптированность** — материал должен соответствовать заявленному уровню сложности.

**Обязательная проверка:**
- Все сгенерированные с использованием ИИ материалы должны быть проверены преподавателем на корректность.
- В случае использования автоматической генерации необходимо выборочно проверять результаты.
- При обнаружении ошибок — внести исправления в исходные данные.

## Примеры использования в разных сценариях

### Сценарий 1: Подготовка к экзамену

1. Используйте ИИ-генерацию для создания 20 вариантов экзаменационных билетов.
2. Проверьте каждый билет на корректность (ручная валидация).
3. Доработайте формулировки слабых вопросов.
4. Сформируйте итоговый набор билетов.

### Сценарий 2: Группа с разным уровнем подготовки

1. Используйте шаблоны для создания базовой версии работы.
2. С помощью ИИ адаптируйте её для слабых и сильных студентов.
3. Проверьте адаптированные версии.
4. Выдайте соответствующие варианты студентам.

### Сценарий 3: Регулярное обновление банка вопросов

1. Подготовьте структурированное описание вопросов в JSON (ручное использование).
2. Настройте автоматическую генерацию тестов из банка.
3. Периодически пополняйте банк через ИИ-генерацию новых вопросов.
4. Проводите тестирование через LMS с автоматической выборкой.
