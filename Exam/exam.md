# Промежуточная аттестация. Экзамен
# МЕТОДИЧЕСКИЕ УКАЗАНИЯ 
## по выполнению, оформлению и защите сквозного проекта по дисциплине «Управление проектами в сфере искусственного интеллекта»

### 1. Общие положения
Настоящие методические указания определяют единый порядок выполнения и защиты итогового проекта. Проект представляет собой комплекс взаимосвязанных документов по управлению AI-проектом, создаваемых в течение всего периода изучения дисциплины.

**Цель проекта:** формирование практических навыков управления жизненным циклом AI-проекта посредством последовательной разработки основных документов проектного управления.

**Принципы выполнения:**
* Единая тема проекта на протяжении всего семестра.
* Последовательное развитие проекта (сквозной характер лабораторных работ).
* Взаимосвязь всех разделов и единая структура документации.

### 2. Этапы выполнения проекта
Проект формируется поэтапно. Каждая лабораторная работа является отдельным этапом, результаты которого используются в последующих разделах.

| Этап | Тема лабораторной работы | Формируемый результат |
| :--- | :--- | :--- |
| 1 | Устав AI-проекта | Паспорт проекта, цели, требования |
| 2 | Управление содержанием | WBS, содержание и границы проекта |
| 3 | Формирование AI-команды | RACI, команда, перечень работ |
| 4 | Планирование сроков | Сетевой график, диаграмма Ганта |
| 5 | Управление стоимостью | Бюджет проекта |
| 6 | Управление качеством | План качества |
| 7 | Управление рисками | Реестр рисков |
| 8 | Выбор методологии | План реализации проекта |
| **Экзамен** | **Защита проекта** | **Итоговая презентация (питч)** |

### 3. Требования к самостоятельности и использованию ИИ
Обучающийся обязан самостоятельно выполнять расчеты, принимать проектные решения и обосновывать выбранные методы. 
**Важно:** Использование генеративного искусственного интеллекта допускается *исключительно* как вспомогательного инструмента. Ответственность за достоверность материалов и адаптацию сгенерированных текстов полностью несет обучающийся.

### 4. Рекомендуемые программные средства
* **RACI-матрица:** Google Docs / MS Excel.
* **Организационная структура:** Miro / Trello / MS Visio (или аналог).
* **Аналитическая записка:** MS Word / Google Docs.
* **Диаграмма Ганта:** MS Project / GanttProject.
* **Сетевая модель и критический путь:** MS Excel / Google Sheets (вручную) или Python (библиотека `networkx`).
* **Презентация:** MS PowerPoint / LibreOffice Impress / Google Slides.

### 5. Требования к оформлению пояснительной записки

#### 5.1. Параметры страницы и шрифт
* **Формат:** А4 (210 × 297 мм), печать на одной стороне.
* **Поля:** левое — 30 мм, правое — 15 мм, верхнее и нижнее — 20 мм.
* **Шрифт:** Times New Roman, 14 pt, цвет черный, начертание обычное.
* **Интервал:** полуторный (1,5).
* **Абзацный отступ:** 1,25 см. Выравнивание текста по ширине.

#### 5.2. Структурные элементы и заголовки
* Элементы («СОДЕРЖАНИЕ», «ВВЕДЕНИЕ», «ЗАКЛЮЧЕНИЕ», «СПИСОК ИСТОЧНИКОВ», «ПРИЛОЖЕНИЯ») начинаются с новой страницы.
* **Нумерация разделов:** арабские цифры (`1 Общая характеристика проекта`). Подразделы: `1.1`, `1.2`. Пункты: `1.2.1`.
* Заголовки разделов — **полужирным шрифтом**, без точки в конце, без подчеркивания. Переносы слов запрещены. После заголовка — одна пустая строка.

#### 5.3. Нумерация, таблицы и рисунки
* **Страницы:** сквозная нумерация арабскими цифрами внизу по центру. Титульный лист включается в нумерацию, но номер на нем не ставится.
* **Таблицы:** нумерация по всему документу (например, *Таблица 4 — План управления рисками*). Название над таблицей слева. Обязательны ссылки в тексте.
* **Рисунки:** нумерация и название *под* рисунком (например, *Рисунок 5 — Диаграмма Ганта проекта*). Обязательны ссылки в тексте.

#### 5.4. Формулы, ссылки и источники
* **Формулы:** по центру строки, нумерация в круглых скобках у правого края. Ниже приводится расшифровка обозначений.
* **Ссылки:** в квадратных скобках (например, `[5]` или `[3, 7, 12]`).
* **Список источников:** оформляется по ГОСТ Р 7.0.100–2018. Не допускаются непроверенные интернет-ресурсы.

#### 5.5. Запрещено
* Отсутствие ссылок на таблицы/рисунки/приложения.
* Использование различных шрифтов без необходимости и декоративного оформления.
* Наличие неотредактированных фрагментов текста, сгенерированных ИИ без проверки автором.
