# КИМ СР. Самостоятельная работа: аналитическое эссе по теме управления ИИ-проектами

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Сквозной (темы распределены по Модулям 1, 2, 3, 4, 6, 7, 8) |
| Вид оценочного средства | Самостоятельная работа — аналитическое эссе |
| Вид контроля | Текущий контроль |
| Рекомендуемый формат выполнения | Индивидуально |
| Рекомендуемая среда | Текстовый редактор (MS Word / Google Docs) |

| Рекомендуемая трудоёмкость | По теме — от 1 до 4 академических часов самостоятельной работы (см. Раздел 5) |

## 2. Назначение КИМ

КИМ предназначен для организации самостоятельной работы обучающихся по темам, которые в РПД дисциплины вынесены на самостоятельное изучение (форма занятия «СР») и распределены по семи модулям курса. В отличие от практических и лабораторных работ, эти темы не отрабатываются на аудиторных занятиях, но являются обязательной частью содержания дисциплины и должны быть освоены самостоятельно с представлением письменного результата.

Каждая тема СР привязана к конкретному модулю, компетенции и индикатору, что позволяет использовать единый формат аналитического эссе для контроля результатов самостоятельной работы на протяжении всего курса.

## 3. Проверяемый результат обучения

Обучающийся самостоятельно изучает вынесенный на СР теоретический материал, соотносит его с содержанием соответствующего модуля дисциплины, критически анализирует источники и представляет результат в виде структурированного аналитического эссе со ссылками на литературу.

## 4. Проверяемые результаты и компетенции

## 6. Ход выполнения

### Часть 1. Выбор темы

1. Выберите одну тему из перечня в Разделе 5 (или получите тему от преподавателя).
2. Определите, как выбранная тема связана с содержанием соответствующего модуля дисциплины (аудиторные лекции, практика, лабораторная работа этого модуля).

### Часть 2. Изучение источников

1. Подберите не менее 3 источников по теме — используйте список рекомендуемой литературы (Раздел 7) и/или самостоятельно найденные источники (научные статьи, отраслевые отчёты, документация методологий и стандартов).
2. Обязательно используйте минимум один источник, непосредственно относящийся к специфике ИИ-проектов (а не общего проектного управления), если тема это предполагает.

### Часть 3. Написание эссе

Структура эссе должна включать:

1. постановку проблемы — почему выбранная тема актуальна для управления ИИ-проектами;
2. краткий обзор подходов/концепций по теме с опорой на источники;
3. анализ применимости рассмотренных подходов к специфике проектов в сфере ИИ (по сравнению с классическими ИТ- или индустриальными проектами);
4. собственный вывод обучающегося — не пересказ источников, а обоснованная позиция.

### Часть 4. Оформление ссылок

Оформите список использованных источников и ссылки на них в тексте эссе в соответствии с требованиями `kim_sr_student_report_template.md`.

## 8. Формат сдачи

Обучающийся сдаёт:

1. текст аналитического эссе (3–5 страниц, без учёта титульного листа и списка литературы);
2. список использованных источников (не менее 3);
3. указание выбранной темы и её номера по перечню Раздела 5.

## 9. Требования к отчёту

Эссе должно содержать:

1. титульную часть с указанием темы, модуля и компетенции;
2. постановку проблемы;
3. обзор подходов/концепций по теме со ссылками на источники;
4. анализ применимости к специфике ИИ-проектов;
5. собственный обоснованный вывод;
6. список использованных источников.

## 10. Контрольные вопросы

1. Почему тема, вынесенная на самостоятельную работу, была включена именно в этот модуль дисциплины?
2. Какие ключевые отличия рассмотренного вами подхода/концепции от практики классических ИТ-проектов вы выявили?
3. Какие источники оказались наиболее информативными и почему?
4. Какой вывод из вашего эссе можно применить на практике при реализации ИИ-проекта?
5. Какие ограничения или спорные моменты рассмотренного подхода вы видите?

## 11. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- поиска и предварительного обзора источников по теме;
- проверки текста эссе (грамматика, структура);
- генерации черновых формулировок, которые обучающийся затем перерабатывает самостоятельно.

Обязательно указывается:

1. какой инструмент использовался;
2. для какой части работы;
3. какие фрагменты были изменены после проверки;
4. как обучающийся убедился в корректности фактов и ссылок (ИИ-инструменты могут искажать библиографические данные — необходима самостоятельная проверка).

Не допускается сдача эссе, полностью сгенерированного ИИ без самостоятельной проверки источников и формулирования собственного вывода.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

Итоговый вклад КИМ в БРС рассчитывается по формуле:

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```

Так как КИМ является сквозным и может засчитываться несколько раз (по числу выбранных тем в течение курса — при таком решении кафедры), способ агрегации нескольких оценок СР в итоговый балл фиксируется отдельно в `docs/evaluation_policy.md`.
