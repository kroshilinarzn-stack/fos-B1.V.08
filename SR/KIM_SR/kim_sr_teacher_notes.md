# Методические указания для преподавателя

## Назначение

Этот КИМ используется для контроля результатов самостоятельной работы (СР) обучающихся по темам, которые в РПД дисциплины «Управление проектами в сфере ИИ» распределены между модулями 1, 2, 3, 4, 6, 7 и 8 (форма занятия «СР»), но не отрабатываются на аудиторных занятиях.

В отличие от модульных КИМ (практические задания, лабораторные работы), данный КИМ — единый шаблон, применяемый многократно в течение курса, по одному разу на каждую (или на выбранные) тему СР.

## Рекомендации по проведению

1. Темы СР можно закреплять за обучающимися как по свободному выбору (см. `kim_sr_task.md`, Раздел 5), так и распределять преподавателем — например, чтобы избежать повторов в группе.
2. Срок сдачи эссе рекомендуется устанавливать вскоре после изучения соответствующего модуля на аудиторных занятиях, чтобы тема была ещё актуальна для обучающегося.
3. Так как объём тем СР по модулям разный (1–4 академических часа), для тем с большим объёмом (например, тема 7 «Выбор методологии для AI-проекта», 4 ч) можно устанавливать более высокие требования к глубине анализа при том же формате эссе.
4. Рекомендуется не проверять более 2–3 эссе от одного обучающегося за модуль, чтобы не перегружать текущий контроль.

## Что проверять в первую очередь

- соответствует ли содержание эссе заявленной теме и модулю;
- использовано ли не менее 3 источников, и относятся ли они по существу к теме;
- есть ли содержательный анализ применимости к специфике ИИ-проектов, а не общие рассуждения о проектном управлении;
- присутствует ли самостоятельный вывод, а не пересказ источников;
- корректно ли оформлены ссылки и список литературы.

## Типичные ошибки студентов

1. Подменяют анализ применимости к ИИ-проектам общими рассуждениями о проектном управлении без ИИ-специфики.
2. Используют только один источник (чаще всего — общедоступную статью в интернете) без сопоставления точек зрения.
3. Не формулируют собственный вывод, ограничиваясь пересказом источников.
4. Оформляют ссылки некорректно или не оформляют вовсе.
5. Сдают текст, по стилю и содержанию похожий на полностью сгенерированный ИИ, без признаков самостоятельной переработки и без раскрытия того, как использовался ИИ-инструмент.

## Минимальная версия задания

Для слабой группы можно:

1. закрепить тему за студентом (без свободного выбора), чтобы сократить время на самоопределение;
2. снизить требование к объёму эссе до 2–3 страниц;
3. допустить использование 2 источников вместо 3 при условии, что оба непосредственно раскрывают тему.

## Расширенная версия задания

Для сильной группы можно:

1. потребовать сравнение минимум двух альтернативных точек зрения по теме из разных источников;
2. добавить требование привести конкретный пример (кейс) применения рассмотренного подхода в реальном ИИ-проекте;
3. организовать короткое устное обсуждение эссе (5 минут) как часть текущего контроля модуля.
