# Рубрика оценивания практического занятия «Подбор методологии разработки AI-проекта (CRISP-DM / CRISP-ML(Q))»

Оценивание проводится по единой 10-балльной шкале ФОС. Перевод результата в БРС выполняется не умножением на 10, а по весу данной работы, который фиксируется в `docs/evaluation_policy.md`.

## 1. Критерии оценивания

| Критерий | Баллы |
|---|---:|
| Полнота анализа характеристик AI-проекта, влияющих на выбор методологии | 1.5 |
| Корректность сравнительного анализа методологий CRISP-DM и CRISP-ML(Q) | 1.5 |
| Обоснованность выбора методологии (матрица взвешенных критериев) | 2.0 |
| Качество адаптации этапов методологии к перечню работ AI-проекта | 2.0 |
| Организация процесса разработки (роли, результаты, точки контроля качества) | 1.5 |
| Анализ рисков и ограничений выбранной методологии | 1.0 |
| Оформление отчёта и качество защиты решения | 0.5 |
| **Итого** | **10.0** |

## 2. Уровни достижения результата

| Уровень | Баллы | Описание |
|---|---:|---|
| Не достигнут | 0–3 | Методология выбрана без анализа характеристик проекта или выбор не обоснован; сравнение методологий отсутствует. |
| Базовый | 4–5 | Проведён анализ отдельных характеристик проекта, методологии сравниваются формально (без матрицы критериев), выбор частично обоснован. |
| Средний | 6–7 | Построена матрица обоснования выбора методологии, этапы методологии сопоставлены с перечнем работ AI-проекта, план организации разработки заполнен не полностью. |
| Высокий | 8–9 | Обоснование выбора методологии полное и аргументированное, адаптация этапов и план организации разработки выполнены корректно, определены точки контроля качества. |
| Продвинутый | 10 | Работа демонстрирует зрелое понимание специфики AI-проекта; выполнено дополнительное исследование (гибридный подход, сопоставление с Agile, анализ чувствительности выбора к изменению условий проекта), даны обоснованные выводы об ограничениях решения. |

## 3. Пересчёт в БРС

Вес работы в балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_работы / 10) × вес_работы
```

Например, если вес работы в БРС составляет 2 балла, а обучающийся получил 7 из 10 по данной рубрике, итоговый вклад работы равен 1,4 балла.

## 4. Типовые основания для снижения баллов

- выбор методологии не опирается на анализ конкретных характеристик AI-проекта;
- сравнение CRISP-DM и CRISP-ML(Q) ограничивается общим теоретическим описанием без привязки к проекту;
- матрица обоснования выбора не содержит весов критериев или расчёта взвешенной суммы;
- этапы выбранной методологии не сопоставлены с реальным перечнем работ AI-проекта;
- план организации разработки не содержит точек контроля качества (quality gates);
- не обоснованы ограничения выбранной методологии и условия, при которых решение может измениться;
- отчёт не структурирован по требованиям раздела 12 `kim_4_practice_task.md`;
- не указано использование генеративного ИИ при подготовке анализа или отчёта.
