# Варианты индивидуализации задания

## Вариант A. Базовый

Построить таблицу операций, сетевую модель и диаграмму Ганта для AI-проекта из 6–8 операций. Рассчитать критический путь и резервы времени.

## Вариант B. Оптимизация «время − затраты»

Для проекта с известной стоимостью ускорения отдельных операций выполнить crashing-анализ и найти минимальную стоимость сокращения срока проекта на заданное число дней.

## Вариант C. Метод критической цепи

Построить план с учётом буферов проекта и питающих буферов по методу критической цепи; сравнить итоговый срок с классическим методом критического пути.

## Вариант D. Сравнение методологий

Сравнить структуру календарного плана для одного и того же AI-проекта при использовании CRISP-DM и CRISP-ML(Q).

## Вариант E. Итеративное планирование

Разработать план в agile-подобном (итеративном) формате со спринтами и сравнить его с классическим сетевым планированием по гибкости к изменениям.

## Вариант F. Каскад изменений

Смоделировать последовательность из двух-трёх изменений (новые данные → доработка признаков → повторное обучение модели) и показать, как каждое из них влияет на критический путь и итоговый срок проекта.
