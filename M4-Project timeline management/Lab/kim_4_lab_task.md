# Практическая работа. Формирование AI-команды проекта

> Работа адаптирует лабораторную работу №1 «Формирование команды проекта» в формат практической работы: диагностика ролевых предпочтений участников, ранжирование ролей и распределение функций внутри AI-команды.

## 1. Паспорт работы

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами |
| Тема | Формирование AI-команды проекта |
| Вид оценочного средства | Практическая работа |
| Вид контроля | Текущий контроль |
| Рекомендуемый формат выполнения | В полном составе AI-команды (5 человек) |
| Исходные данные | Результаты тестирования участников по 4 методикам (Приложения 1–4) |
| Основной артефакт | Итоговая матрица распределения AI-ролей + отчёт с обоснованием |

## 2. Цель работы

Сформировать сбалансированную AI-команду из 5 человек на основе комплексной диагностики ролевых предпочтений, провести ранжирование ролей для каждого участника и распределить командные функции внутри AI-команды в соответствии с полученными результатами.

## 3. Задачи

1. Провести индивидуальную диагностику каждого участника AI-команды по четырём методикам:
   - Р.М. Белбин (командные роли);
   - DISC (поведенческие стили);
   - Р.О. Немов (групповые роли);
   - С. Деллингер (психогеометрическая типология).
2. Обработать результаты и определить:
   - доминирующие роли (наивысшие показатели);
   - субдоминирующие роли (поддерживающие, вторые по значимости);
   - зоны развития (наименьшие показатели, «анти-роли»).
3. Составить для каждого участника ранжированный ролевой профиль (от 1 до 3–5 ролей в порядке убывания выраженности).
4. Построить гистограммы распределения ролей в AI-команде по каждой методике.
5. На основе индивидуальных ранжированных профилей сформировать итоговую матрицу распределения ролей в AI-команде из 5 человек.
6. Обеспечить функциональную полноту AI-команды (см. раздел 6).
7. Обосновать принятые решения и представить отчёт.

## 4. Проверяемый результат обучения

Участник умеет проходить и интерпретировать диагностические методики ролевых предпочтений, ранжировать роли, участвовать в согласованном распределении функций внутри AI-команды и обосновывать итоговое распределение ролей.

## 5. Ход выполнения

### Шаг 1. Индивидуальное тестирование

Каждый участник AI-команды самостоятельно проходит четыре теста (Приложения 1–4) и фиксирует первичные ответы в соответствии с инструкциями к каждой методике.

### Шаг 2. Обработка результатов по каждой методике

**Методика 1. Р.М. Белбин.**
1. Пройти тест (Приложение 1): в каждом из 7 разделов распределить 10 баллов между вариантами ответа согласно тому, насколько они подходят собственному поведению в AI-команде.
2. Определить свою роль (роли) по ключу теста и заполнить форму «Распределение ролей в AI-команде».
3. Построить гистограмму с промежуточными результатами.
4. Ранжировать и распределить роли в ранжированный список.

Ключ интерпретации баллов: до 3 баллов — случайное попадание (не учитывается); 8–11 — роль выражена в незначительной степени; 12–19 — в средней степени; свыше 20 — в высокой степени. Участник может одновременно занимать 2–3 роли в соответствии со степенью выраженности.

**Методика 2. DISC.**
1. Пройти тест (Приложение 2) и определить свою роль (роли) по ключу.
2. Заполнить форму «Распределение ролей в AI-команде».
3. Построить гистограмму с промежуточными результатами.
4. Ранжировать и распределить роли.

Доминирующий поведенческий стиль — тот, что набрал наибольшее количество очков; вторичный стиль — набравший столько же или на 1–2 очка меньше.

**Методика 3. Р.О. Немов.**
1. Пройти тест (Приложение 3), распределяя в каждом из 7 разделов 10 баллов между утверждениями.
2. Построить промежуточную таблицу по ключу-образцу и найти сумму баллов по каждой из 8 ролей (I–VIII).
3. Выделить роли с наибольшими суммами — это роли, которые участник чаще выполняет в группе.
4. Построить гистограмму с промежуточными результатами и ранжировать роли.

**Методика 4. С. Деллингер.**
1. Пройти тест (Приложение 4): выбрать из пяти фигур (квадрат, треугольник, прямоугольник, круг, зигзаг) наиболее близкую и проранжировать все пять по степени предпочтения (от 1 до 5).
2. Определить свою роль (роли) на основе интерпретации доминирующей и дополнительных фигур.
3. Построить гистограмму с промежуточными результатами и ранжировать роли.

### Шаг 3. Построение гистограмм по AI-команде

Для каждой методики отдельно построить гистограмму, отражающую количество участников AI-команды, у которых данная роль/стиль/фигура является доминирующей.

Анализ гистограммы:
- выявить «перегруженные» роли (3 и более человека с одной ролью);
- выявить «дефицитные» роли (роль отсутствует у всех участников);
- сделать предварительный вывод о несбалансированности AI-команды.

### Шаг 4. Составление индивидуального ранжированного профиля

Для каждого участника AI-команды заполняется таблица «Индивидуальный ранжированный профиль» со столбцами: ранг, роль по Белбину, стиль DISC, роль по Немову, фигура по Деллингеру, интегральное описание.

**Правила ранжирования:**

1. Ранг 1 — роль, которая имеет наивысший балл по Белбину (>20, либо 12–19 при отсутствии баллов свыше 20), совпадает или коррелирует с доминирующим стилем DISC, совпадает с ролью-лидером по Немову и с фигурой №1 по Деллингеру (либо фигурой №2 при комбинированном типе).
2. Ранг 2 — субдоминанта по Белбину, подтверждённая хотя бы одной другой методикой.
3. Ранг 3 и далее — роли со средними баллами, которые участник может исполнять при необходимости, а также роли, соответствующие фигурам №2–4 по Деллингеру.
4. Фигура №5 по Деллингеру и роль с наименьшим баллом по Белбину выносятся в отдельную графу «Зона развития/стресса» и не рекомендуются к назначению в качестве основной.

### Шаг 5. Формирование итоговой матрицы распределения ролей в AI-команде

**Критерии распределения:**

1. Каждому участнику назначается одна основная роль, соответствующая его Рангу 1.
2. При конфликте (два участника претендуют на одну и ту же основную роль) решение принимается на основе: большего балла по Белбину; большего числа подтверждений по другим методикам; добровольного согласия одного из участников перейти на поддерживающую роль.
3. В AI-команде из 5 человек должны быть представлены все 5 функциональных блоков (см. раздел 6).
4. Допускается совмещение блоков, если у участника в Ранге 2 присутствует роль из недостающего блока.

## 6. Функциональная полнота AI-команды

| Блок | Содержание | Соответствующие роли |
|---|---|---|
| Блок А. Лидерство и координация | Целеполагание, координация работы AI-команды | Координатор / Председатель / D / Треугольник |
| Блок Б. Генерация идей | Креатив, новые решения для AI-проекта | Генератор идей / Зигзаг / I |
| Блок В. Аналитика и контроль | Критика, оценка идей, контроль качества | Аналитик / Оценщик идей / C / Квадрат |
| Блок Г. Коммуникации и ресурсы | Внешние связи, привлечение ресурсов | Дипломат / Исследователь ресурсов / Круг |
| Блок Д. Исполнение и завершение | Дисциплина, доведение работ AI-проекта до конца | Исполнитель / Финишер / Организатор работы |

## 7. Подготовка отчёта

Отчёт AI-команды должен содержать:

1. титульный лист с указанием темы, группы и состава AI-команды;
2. индивидуальные протоколы каждого участника (сырые данные по 4 тестам);
3. сводные таблицы и гистограммы по AI-команде (по каждой методике);
4. индивидуальные ранжированные профили (на каждого участника);
5. итоговую матрицу распределения ролей в AI-команде;
6. обоснование принятых решений: почему участник получил ту или иную роль; как разрешён конфликт ролей (если был); как обеспечена функциональная полнота AI-команды; какие роли оказались в дефиците и за счёт чего этот дефицит покрыт;
7. вывод о готовности AI-команды к выполнению AI-проекта.

## 8. Требования к оформлению

1. Отчёт выполняется на листах формата А4.
2. Таблицы и гистограммы нумеруются и подписываются.
3. Гистограммы выполняются в электронном виде.
4. Все выводы аргументируются ссылками на конкретные числовые значения тестов.

## 9. Приложения

Приложения 1–4 — полные тексты диагностических методик (Р.М. Белбин, DISC, Р.О. Немов, С. Деллингер) с ключами обработки и интерпретацией результатов — прилагаются к практической работе отдельным файлом.

## 10. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- пояснения методик обработки результатов тестов;
- проверки корректности расчётов сумм баллов;
- проверки текста обоснования и отчёта.

Обязательно указывается: какой инструмент использовался; для какой части работы; какие фрагменты были изменены после проверки; как участник AI-команды убедился в корректности результата.

Не допускается формирование итоговой матрицы распределения ролей исключительно на основе рекомендаций ИИ без реального прохождения тестов всеми участниками AI-команды.

## Пересчёт в БРС

Работа оценивается по единой 10-балльной шкале ФОС. Вес данной работы в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_работы / 10) × вес_работы
```
