# КИМ 4. Управление сроками проекта

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Модуль 4. Управление сроками проекта |
| Вид оценочного средства | Практическое задание + лабораторная работа |
| Вид контроля | Текущий контроль |
| Рекомендуемый формат выполнения | Индивидуально или в малой команде |
| Рекомендуемая среда | MS Project / GanttProject / Excel / Google Sheets |
| Основной артефакт | Сетевая модель проекта, диаграмма Ганта с расчётом критического пути + аналитическая записка |
| Рекомендуемая трудоёмкость | 8 академических часов (лекции, семинар-практикум, лабораторная работа, самостоятельная работа) |

## 2. Назначение КИМ

КИМ предназначен для проверки того, что обучающийся умеет определять состав и взаимосвязи операций проекта, строить сетевые модели, выполнять календарное планирование, рассчитывать критический путь и резервы времени, а также корректировать план проекта в условиях неопределённости, характерной для AI-проектов (изменение требований, появление новых данных, изменение результатов обучения моделей).

## 3. Проверяемый результат обучения

Обучающийся осуществляет ведение проекта в области ИИ, планирование и контроль задач, оценку ресурсов; подбирает методологию (CRISP-DM, CRISP-ML(Q)) и организует процесс разработки.

## 4. Проверяемые результаты и компетенции

| Компетенция / область результата | Ориентировочный индикатор | Уровень | Как проявляется в задании |
|---|---|---|---|
| LC-4 | 4.1 | Базовый/Средний | Определение состава операций проекта, построение сетевой модели и календарного плана, расчёт критического пути; корректировка плана в условиях неопределённости AI-проекта. |

## 5. Исходные данные (кейс проекта)

Для выполнения задания используется учебный кейс AI-проекта (может совпадать с кейсом из модуля 3 либо быть новым), содержащий:

1. перечень основных этапов разработки по выбранной методологии (например, CRISP-DM: бизнес-понимание, понимание данных, подготовка данных, моделирование, оценка, внедрение);
2. ориентировочную длительность этапов и доступные ресурсы;
3. известные ограничения по срокам (например, дедлайн демонстрации MVP заказчику).

## 6. Программные средства

Рекомендуемый стек:

- MS Project / GanttProject — для построения диаграммы Ганта;
- MS Excel / Google Sheets — для расчёта сетевой модели и критического пути вручную;
- при необходимости — Python (например, библиотека `networkx`) для автоматизации расчёта критического пути.

## 7. Ход выполнения

### Часть 1. Определение состава и взаимосвязей операций проекта

1. Разбейте выбранный AI-проект на операции (задачи) в соответствии с этапами методологии (CRISP-DM/CRISP-ML(Q) или иной).
2. Для каждой операции определите длительность и предшествующие операции (зависимости).
3. Оформите таблицу операций проекта.

### Часть 2. Построение сетевой модели

1. Постройте сетевую модель проекта (диаграмму типа «работы-вершины» или «работы-дуги»).
2. Рассчитайте ранние и поздние сроки начала/окончания операций.
3. Определите критический путь и резервы времени для некритических операций.

### Часть 3. Календарное планирование и диаграмма Ганта

1. Постройте диаграмму Ганта с учётом рассчитанных сроков и зависимостей задач.
2. Отметьте на диаграмме критический путь.
3. При наличии данных о стоимости ускорения отдельных операций рассмотрите оптимизацию «время − затраты» (crashing) для сокращения длительности проекта.

### Часть 4. Управление сроками в условиях неопределённости

1. Опишите минимум два источника неопределённости, характерных для AI-проектов (например, непредсказуемая длительность экспериментов по обучению модели, изменение объёма/качества доступных данных, изменение требований заказчика по мере получения промежуточных результатов).
2. Предложите способы учёта этой неопределённости в календарном плане (например, буферы времени, итеративное планирование, метод критической цепи, переоценка оценок по мере поступления новой информации).

### Часть 5. Корректировка плана AI-проекта

1. Смоделируйте ситуацию изменения требований, появления новых данных или изменения результатов обучения модели (например, модель не достигла целевой метрики качества и требуется дополнительная итерация).
2. Скорректируйте сетевую модель и диаграмму Ганта с учётом этого изменения.
3. Опишите, как изменился критический путь и итоговый срок проекта.

### Часть 6. Дополнительное исследование

Выполните минимум одно дополнительное исследование:

- сравните классическое и итеративное (agile-подобное) планирование сроков для выбранного AI-проекта;
- рассчитайте, как изменение оценки длительности одной из операций (например, из-за неопределённости в экспериментах) влияет на критический путь;
- предложите буферную стратегию (например, по методу критической цепи) для управления сроками;
- постройте сравнение двух вариантов методологии (CRISP-DM vs CRISP-ML(Q)) с точки зрения влияния на календарный план.

## 8. Формат сдачи

Обучающийся сдаёт:

1. таблицу операций проекта с длительностями и зависимостями;
2. сетевую модель проекта с расчётом критического пути и резервов времени;
3. диаграмму Ганта;
4. аналитическую записку с описанием источников неопределённости и корректировки плана;
5. (при выполнении) материалы дополнительного исследования.

## 9. Требования к отчёту

Отчёт должен содержать:

1. описание кейса AI-проекта и выбранной методологии разработки;
2. таблицу операций с длительностями и зависимостями;
3. сетевую модель с расчётом критического пути и резервов времени;
4. диаграмму Ганта;
5. анализ источников неопределённости сроков в AI-проекте;
6. описание корректировки плана при изменении требований/данных/результатов моделирования;
7. вывод о ключевых рисках управления сроками в выбранном проекте.

## 10. Контрольные вопросы

1. Что такое сетевая модель проекта и какие виды сетевых моделей существуют?
2. Как рассчитываются ранние и поздние сроки начала и окончания операций?
3. Что такое критический путь и почему он важен для управления сроками?
4. Что такое резерв времени операции и как он рассчитывается?
5. В чём заключается оптимизация «время − затраты» (crashing)?
6. Какие источники неопределённости характерны именно для AI-проектов по сравнению с классическими проектами?
7. Что такое метод критической цепи и как он помогает управлять неопределённостью сроков?
8. Как выбор методологии (CRISP-DM, CRISP-ML(Q)) влияет на структуру календарного плана?
9. Как изменение результатов обучения модели может повлиять на критический путь проекта?
10. Какие риски возникают при игнорировании неопределённости при планировании сроков AI-проекта?

## 11. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- пояснения методов расчёта критического пути;
- генерации черновой структуры таблицы операций;
- проверки текста аналитической записки.

Обязательно указывается:

1. какой инструмент использовался;
2. для какой части работы;
3. какие фрагменты были изменены после проверки;
4. как обучающийся убедился в корректности расчётов (например, путём ручной проверки критического пути).

Не допускается сдача расчётов критического пути, полностью выполненных ИИ без самостоятельной проверки корректности.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

Итоговый вклад КИМ в БРС рассчитывается по формуле:

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```

Такой подход позволяет использовать единую рубрику для всех КИМ, но не фиксировать внутри отдельного задания общий вес текущего контроля, проекта и экзамена.
