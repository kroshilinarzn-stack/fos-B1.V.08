# КИМ 7. Практическое занятие "Управление стейкхолдерами"

# Методические указания для преподавателя

## Назначение

Практическое занятие формирует компетенцию, связанную с выявлением и классификацией заинтересованных сторон ИИ-проекта, анализа их ожиданий и влияния, разработки стратегий взаимодействия и плана коммуникаций. Работа позволяет оценить способность обучающегося учитывать человеческий фактор при организации процесса разработки и принимать обоснованные управленческие решения.

---

### Рекомендации по проведению

1. Занятие рекомендуется проводить после изучения теоретических основ управления стейкхолдерами.
2. Рекомендуемая форма проведения — практическая работа в малых группах (2–4 человека) с последующей защитой результатов. Допускается индивидуальное выполнение.
3. В качестве исходных данных рекомендуется использовать учебный кейс ИИ-проекта, знакомый обучающимся по предыдущим занятиям, либо предложить выбор из нескольких кейсов с разной спецификой состава стейкхолдеров.
4. Рекомендуемое время на выполнение работы — 2 академических часа аудиторной работы с самостоятельным оформлением отчёта.
5. Для групповой работы рекомендуется распределить роли: менеджер проекта, аналитик, представитель заказчика, технический специалист. Это позволяет обучающимся глубже понять позиции разных стейкхолдеров.

---

## Что проверять в первую очередь

- идентифицированы ли все ключевые группы стейкхолдеров (внутренние и внешние);
- указана ли роль и характер участия каждого стейкхолдера;
- проведён ли анализ ожиданий и выявлены ли потенциальные конфликты интересов;
- корректно ли построена матрица «влияние — интерес» с обоснованием оценок;
- соответствуют ли стратегии взаимодействия квадрантам матрицы;
- содержит ли план коммуникаций все обязательные элементы (формат, периодичность, ответственный, канал);
- есть ли содержательный вывод о достаточности коммуникационных мероприятий.

---

## Типичные ошибки студентов

1. Перечень стейкхолдеров ограничивается только заказчиком и командой разработки, без учёта внешних групп (регуляторы, пользователи, поставщики данных).
2. Анализ ожиданий отсутствует или содержит общие формулировки без привязки к конкретному проекту.
3. Оценки влияния и интереса проставлены без аргументации, противоречат описанию роли стейкхолдера.
4. Матрица построена не по двухмерной шкале, а в виде простого перечня стейкхолдеров с текстовым описанием.
5. Стратегии взаимодействия не соответствуют квадрантам матрицы (например, для стейкхолдеров из первого квадранта выбрана стратегия наблюдения).
6. План коммуникаций не содержит периодичности или ответственных за реализацию.
7. План коммуникаций не учитывает этапы жизненного цикла ИИ-проекта и изменение информационных потребностей стейкхолдеров с течением времени.
8. Отчёт содержит матрицу и таблицы, но отсутствует пояснительный текст, раскрывающий логику принятых решений.

---

### Минимальная версия задания

Для слабой группы можно сократить задание до:

1. составления перечня из 6–8 ключевых стейкхолдеров с указанием их ролей;
2. построения матрицы «влияние — интерес» без детального обоснования оценок;
3. определения стратегий взаимодействия для каждой группы без разработки плана коммуникаций.

---

### Расширенная версия задания

Для сильной группы можно добавить:

1. проведение ролевой игры, в которой обучающиеся поочерёдно выступают в роли разных стейкхолдеров и обосновывают свои интересы;
2. анализ динамики изменения влияния и интереса стейкхолдеров на разных этапах жизненного цикла ИИ-проекта с построением временной шкалы вовлечённости;
3. разработку сценариев реагирования на изменение позиции ключевых стейкхолдеров (например, смена заказчика, ужесточение регуляторных требований);
4. сравнение разработанного плана коммуникаций с эталонными практиками управления стейкхолдерами в ИИ-проектах (на основе кейсов из индустрии);
5. подготовку презентации для защиты перед комиссией, где обучающиеся выступают в роли менеджера проекта, докладывающего стратегию управления стейкхолдерами руководству.
