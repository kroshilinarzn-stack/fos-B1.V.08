# КИМ 7. Коллоквиум по дисциплине «Управление проектами в сфере ИИ» (модуль 7)

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Модуль 7. Управление рисками и взаимодействием в проекте |
| Вид оценочного средства | Коллоквиум (опрос по банку теоретических и проблемных вопросов) |
| Вид контроля | Текущий контроль |
| Рекомендуемый формат выполнения | Индивидуально |
| Рекомендуемая среда | LMS Moodle (банк вопросов типа «эссе») либо устный опрос с использованием того же банка вопросов |
| Основной артефакт | Письменные (в Moodle) или устные ответы на вопросы двух типов: теоретические и проблемные (кейсы) |
| Рекомендуемая трудоёмкость | 1 академический час |

## 2. Назначение КИМ

КИМ предназначен для проверки усвоения материала модуля 7: видов и источников рисков проекта, ведения реестра рисков, управления проблемами и изменениями, взаимодействия с внешними участниками проекта, а также коммуникаций и работы со стейкхолдерами в сложной проектной среде. Коллоквиум проверяет как владение терминологией (теоретический блок), так и умение применять её для анализа проблемных ситуаций, специфичных для управления рисками AI-проектов (блок кейсов).

## 3. Проверяемые результаты обучения и компетенции

| Компетенция | Индикатор | Уровень | Как проявляется в задании |

| LC-4 | 4.2 | Средний | Управление коммуникациями и стейкхолдерами в сложной проектной среде; взаимодействие с внешними участниками проекта. |

## 4. Источники информации

Для подготовки к коллоквиуму обучающиеся используют материалы лекций, лабораторной и практической работ модуля 7, а также источники, указанные в паспортах модульных КИМ дисциплины «Управление проектами в сфере ИИ».

## 5. Структура банка вопросов

Банк вопросов состоит из **двух групп**, разделённых по типу проверяемого результата. Внутри каждой группы вопросы разбиты на **15 тематических подгрупп**, соответствующих содержанию модуля 7:

| Тип вопросов | Назначение | Тип вопроса (Moodle) | Расположение |
|---|---|---|---|
| Теоретические | Проверка владения понятийным аппаратом («дайте определение понятия…») | essay (открытый ответ, оценивается вручную по эталонному определению) | `Question_bank_theory/kim_7_Colloq_theory_gr1.xml` … `gr15.xml` |
| Проблемные (кейсы) | Проверка умения применять понятия и методы для анализа и решения проблемных ситуаций AI-проекта | essay (открытый развёрнутый ответ, оценивается вручную по критериям) | `Question_bank_cases/kim_7_Colloq_case_gr1.xml` … `gr15.xml` |

| № группы | Тема группы |
|---:|---|
| gr1 | Понятие и природа риска проекта |
| gr2 | Источники рисков: предприятие и предметная область |
| gr3 | Источники рисков: ресурсы и техника |
| gr4 | Источники рисков: рынок и контрагенты |
| gr5 | Источники рисков: управление проектом |
| gr6 | Реестр рисков и его ведение |
| gr7 | Управление проблемами проекта |
| gr8 | Управление изменениями проекта |
| gr9 | Управление взаимодействием с внешними участниками |
| gr10 | Стейкхолдеры проекта: идентификация и классификация |
| gr11 | План управления коммуникациями |
| gr12 | Коммуникации в сложной проектной среде |
| gr13 | Идентификация рисков, специфичных для AI-проектов |
| gr14 | Качественная и количественная оценка рисков AI-проекта |
| gr15 | Разработка плана реагирования и реестр TOP-10 |

В теоретической группе каждый файл содержит 1–2 вопроса на определение ключевых понятий темы. В группе кейсов каждый файл содержит проблемную ситуацию, требующую развёрнутого ответа с обоснованием решения.

## 6. Формирование варианта коллоквиума

1. Из случайно выбранных подгрупп (рекомендуется не менее 5 из 15) выбирается один теоретический вопрос и одна проблемная ситуация.
2. Типовой вариант коллоквиума включает 5–7 теоретических вопросов и 5–7 кейсов, охватывающих основное содержание модуля 7.
3. При проведении в формате Moodle-квиза случайная выборка настраивается через категории вопросов, соответствующие файлам `gr{N}.xml`; из каждой выбранной категории берётся один случайный вопрос.
4. При проведении в устном формате вопросы того же банка используются для формирования билетов (см. `kim_7_Colloq_var.md`, вариант «Устное собеседование»).

## 7. Требования к процедуре проведения

### Условия проведения (формат Moodle)

- коллоквиум проводится в системе Moodle (или аналогичной LMS) с использованием банка вопросов типа «эссе»;
- ответы на вопросы проверяются преподавателем вручную по эталонным определениям (теоретический блок) и по критериям оценивания (блок кейсов), указанным в поле `graderinfo` каждого вопроса;
- рекомендуемое время выполнения — 3–4 минуты на теоретический вопрос, 8–10 минут на кейс.

### Условия проведения (устный формат)

- обучающийся получает билет из банка вопросов (см. раздел 6) и время на подготовку (10 минут);
- устный ответ включает раскрытие определений и разбор решения проблемных ситуаций;
- преподаватель вправе задавать уточняющие вопросы по любой из трёх тем модуля.

## 8. Требования к ответу

Ответ на теоретический вопрос должен:

1. раскрывать смысл понятия своими словами (дословное совпадение с эталонной формулировкой не требуется);
2. быть точным и не содержать подмены понятия смежным термином (например, риска — уже наступившей проблемой).

Ответ на кейс должен:

1. демонстрировать понимание контекста проблемной ситуации;
2. содержать системный и полный анализ, учитывающий специфику рисков AI-проектов;
3. предлагать обоснованное и реалистичное решение, включая конкретные управленческие действия.

## 9. Процедура оценивания

Оценивание описано в `kim_7_Colloq_mark.md`. Итоговый балл коллоквиума складывается из суммы баллов за теоретические вопросы (максимум 2 балла за вопрос) и кейсы (максимум 5 баллов за кейс) выбранного варианта с последующим приведением к единой 10-балльной шкале ФОС.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```
