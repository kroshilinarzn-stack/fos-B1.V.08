# Рубрика оценивания лабораторной работы «Реестр рисков TOP-10»

Оценивание проводится по единой 10-балльной шкале ФОС. Перевод результата в БРС выполняется не умножением на 10, а по весу данной работы, который фиксируется в `docs/evaluation_policy.md`.

## 1. Критерии оценивания

| Критерий | Баллы |
|---|---:|
| Полнота идентификации рисков (не менее 15, охват всех категорий, формулировка в виде событий) | 1.5 |
| Корректность фильтрации, группировки и обоснование исключения рисков | 1.0 |
| Обоснованность оценок вероятности и влияния (аргументация, учёт компонентов ущерба) | 1.5 |
| Корректность построения матрицы рисков с выделением цветовых зон и пояснениями | 1.5 |
| Точность расчёта RPN и обоснованность отбора TOP-10 рисков | 1.5 |
| Обоснованность выбора стратегий управления для каждого риска TOP-10 | 1.0 |
| Качество плана реагирования (конкретные меры, ответственные, триггеры) | 1.0 |
| Полнота и качество итогового отчёта (структура, выводы, оформление) | 1.0 |
| **Итого** | **10.0** |

## 2. Уровни достижения результата

| Уровень | Баллы | Описание |
|---|---:|---|
| Не достигнут | 0–3 | Сетевой график не построен или содержит грубые ошибки (циклы, изолированные работы); расчёт временных параметров отсутствует. |
| Базовый | 4–5 | Сетевой график построен, рассчитаны ES/EF, но LS/LF и резервы времени рассчитаны с ошибками или не рассчитаны; критический путь не обоснован. |
| Средний | 6–7 | Все временные параметры рассчитаны корректно, критический путь определён верно, построена диаграмма Ганта. |
| Высокий | 8–9 | Расчёты полностью корректны и воспроизводимы, диаграмма Ганта отражает резервы времени, проведён анализ загрузки ресурсов с обоснованными предложениями по оптимизации. |
| Продвинутый | 10 | Работа полностью воспроизводима; выполнено дополнительное исследование (сравнение нотаций, краш-анализ, анализ чувствительности сроков), даны содержательные выводы о рисках и возможностях сокращения длительности AI-проекта. |

## 3. Пересчёт в БРС

Вес работы в балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_работы / 10) × вес_работы
```

Например, если вес работы в БРС составляет 2 балла, а обучающийся получил 8 из 10 по данной рубрике, итоговый вклад работы равен 1,6 балла.

## 4. Типовые основания для снижения баллов

- в перечне работ отсутствуют зависимости для одной или нескольких работ;
- сетевой график содержит циклы или изолированные работы, не выявленные обучающимся;
- ошибки в расчёте ES/EF при прямом проходе по сети;
- ошибки в расчёте LS/LF при обратном проходе по сети;
- резервы времени рассчитаны неверно или не рассчитаны вовсе;
- критический путь определён без проверки нулевого резерва работ;
- диаграмма Ганта не отражает критический путь или резервы времени;
- отсутствует анализ загрузки ресурсов и предложения по оптимизации;
- расчёты не воспроизводятся (нет исходных данных, формул или файла расчёта);
- не указано использование генеративного ИИ при подготовке расчётов или отчёта.
