# Рубрика оценивания лабораторной работы «Реестр рисков TOP-10»

Оценивание проводится по единой 10-балльной шкале ФОС. Перевод результата в БРС выполняется не умножением на 10, а по весу данной работы, который фиксируется в `docs/evaluation_policy.md`.

## 1. Критерии оценивания

| Критерий | Баллы |
|---|---:|
| Полнота идентификации рисков (не менее 15, охват всех категорий, формулировка в виде событий) | 1.5 |
| Корректность фильтрации, группировки и обоснование исключения рисков | 1.0 |
| Обоснованность оценок вероятности и влияния (аргументация, учёт компонентов ущерба) | 1.5 |
| Корректность построения матрицы рисков с выделением цветовых зон и пояснениями | 1.5 |
| Точность расчёта RPN и обоснованность отбора TOP-10 рисков | 1.5 |
| Обоснованность выбора стратегий управления для каждого риска TOP-10 | 1.0 |
| Качество плана реагирования (конкретные меры, ответственные, триггеры) | 1.0 |
| Полнота и качество итогового отчёта (структура, выводы, оформление) | 1.0 |
| **Итого** | **10.0** |

## 2. Уровни достижения результата

| Уровень | Баллы | Описание |
|---|---:|---|
| **Не достигнут** | 0–3 | Идентифицировано менее 10 рисков, отсутствует группировка по категориям. Оценки вероятности и влияния не проставлены либо проставлены без обоснования. Матрица рисков не построена или построена с грубыми ошибками. RPN не рассчитан. Реестр TOP-10 не сформирован. |
| **Базовый** | 4–5 | Выявлено 10–12 рисков, проведена группировка по категориям. Оценки P и I проставлены, но аргументация отсутствует или недостаточна. Матрица рисков построена, но цветовые зоны не выделены или не пояснены. RPN рассчитан с ошибками. Реестр TOP-10 сформирован, но стратегии управления не обоснованы. |
| **Средний** | 6–7 | Выявлено 13–15 рисков с распределением по категориям. Оценки P и I обоснованы. Матрица рисков построена корректно, зоны выделены и пояснены. RPN рассчитан верно, TOP-10 отобран корректно. Для каждого риска TOP-10 выбрана стратегия управления, но планы реагирования содержат общие формулировки без указания ответственных и триггеров. |
| **Высокий** | 8–9 | Выявлено 15 и более рисков, охватывающих все категории. Оценки P и I аргументированы, влияние оценено с учётом нескольких компонентов ущерба. Матрица рисков построена качественно, цветовые зоны обоснованы. RPN рассчитан без ошибок, TOP-10 обоснован и согласован с матрицей. Для каждого риска выбрана адекватная стратегия с пояснением. Разработаны конкретные планы реагирования с указанием мер, ответственных ролей и триггеров. Отчёт имеет чёткую структуру и содержит обоснованные выводы. |
| **Продвинутый** | 10 | Все требования высокого уровня выполнены. Дополнительно проведён анализ чувствительности рисков (изменение RPN при вариации оценок), предложены альтернативные стратегии для критических рисков с оценкой их эффективности. Выполнено сравнение полученного реестра с типовыми рисками аналогичных ИИ-проектов. Сформулированы рекомендации по интеграции риск-менеджмента в процесс управления проектом на всех стадиях жизненного цикла. |

## 3. Пересчёт в БРС

Вес работы в балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_работы / 10) × вес_работы
```

Например, если вес работы в БРС составляет 2 балла, а обучающийся получил 8 из 10 по данной рубрике, итоговый вклад работы равен 1,6 балла.

## 4. Типовые основания для снижения баллов

- в перечне работ отсутствуют зависимости для одной или нескольких работ;
- сетевой график содержит циклы или изолированные работы, не выявленные обучающимся;
- ошибки в расчёте ES/EF при прямом проходе по сети;
- ошибки в расчёте LS/LF при обратном проходе по сети;
- резервы времени рассчитаны неверно или не рассчитаны вовсе;
- критический путь определён без проверки нулевого резерва работ;
- диаграмма Ганта не отражает критический путь или резервы времени;
- отсутствует анализ загрузки ресурсов и предложения по оптимизации;
- расчёты не воспроизводятся (нет исходных данных, формул или файла расчёта);
- не указано использование генеративного ИИ при подготовке расчётов или отчёта.
