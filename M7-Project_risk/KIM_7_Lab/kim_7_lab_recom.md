# КИМ 7. Лабораторная работа "Реестр рисков TOP-10"

# Методические указания для преподавателя

## Назначение

Работа продолжает цикл лабораторных работ по управлению AI-проектом: после формирования AI-команды и перечня работ и плана управления качеством AI-проекта обучающиеся переходят к выявлению рисков, оценке степени влияния и вероятности возникновения, выработке стратегии реагирования.

## Рекомендации по проведению

1. До занятия рекомендуется убедиться, что у каждого обучающегося (или AI-команды) есть готовый сводный перечень работ с зависимостями из предыдущих лабораторных работ.
2. На занятии целесообразно сначала разобрать вручную небольшой пример (5–7 рисков), чтобы студенты усвоили порядок выявления рисков, оценки их влияния и вероятности, прежде чем переходить к полному перечню рисков.
3. Построение матрицы рисков рекомендуется выполнить под контролем преподавателя.
4. Формирование итогового реестра и стратегии реагирования можно оставить самостоятельным, но с промежуточной проверкой минимум одного примера у доски.
5. Анализ и обсуждение итоговых отчетов производится на защите.

## Что проверять в первую очередь

- содержит ли реестр рисков не менее 15 позиций до фильтрации;
- охвачены ли все основные категории рисков (данные, модель, технические, организационные, внешние);
- сформулированы ли риски как конкретные события, а не как общие проблемы;
- проведена ли первичная фильтрация списка (исключены дублирующиеся и малозначимые риски);
- обоснованы ли оценки вероятности и влияния для каждого риска;
- учтены ли несколько компонентов ущерба при оценке влияния (финансовые, временные, качественные, репутационные потери);
- построена ли матрица рисков с выделением цветовых зон;
- рассчитан ли RPN без арифметических ошибок;
- совпадает ли состав реестра TOP-10 с рисками из красной зоны матрицы;
- для каждого риска из TOP-10 определена ли стратегия управления и обоснован ли её выбор;
- содержат ли планы реагирования конкретные меры, ответственных ролей и триггеры запуска;
- содержит ли итоговый отчёт все обязательные разделы;
- обоснован ли вывод о готовности проекта к старту.

---

## Типичные ошибки студентов

1. В исходном списке рисков менее 15 позиций, что не позволяет провести качественную фильтрацию и отбор TOP-10.
2. Риски сформулированы как проблемы, а не как события («низкое качество данных» вместо «доля пропусков в данных превышает 10 %»).
3. Оценки вероятности и влияния проставлены без аргументации или противоречат описанию риска.
4. Влияние риска оценено только по одному компоненту (например, только финансовые потери) без учёта временных, качественных и репутационных последствий.
5. При построении матрицы рисков цветовые зоны не соответствуют значениям вероятности и влияния либо границы зон не пояснены.
6. RPN рассчитан с арифметическими ошибками либо для части рисков расчёт пропущен.
7. В реестр TOP-10 не попали риски из красной зоны матрицы, при этом данное противоречие не пояснено.
8. Для всех рисков TOP-10 выбрана одна и та же стратегия (чаще всего «снижение») без обоснования, почему не применимы другие стратегии.
9. Планы реагирования содержат общие формулировки («улучшить качество данных», «усилить контроль») без указания конкретных мер, ответственных и триггеров.
10. Итоговый отчёт не содержит одного из обязательных разделов (введение, реестр, матрица, анализ красной зоны, вывод).
11. Вывод о готовности проекта к старту не обоснован или противоречит выполненному анализу (например, проект признаётся готовым при наличии трёх рисков в красной зоне без планов реагирования).
12. Расчёты представлены без исходных данных и формул, что делает их невоспроизводимыми.

---

## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. перечня из 10–12 рисков с распределением по категориям;
2. оценки вероятности и влияния по упрощённой шкале (например, трёхбалльной: низкая, средняя, высокая);
3. построения матрицы рисков с выделением зон (без строгого соблюдения числовых границ);
4. расчёта RPN для каждого риска и отбора TOP-5 (вместо TOP-10);
5. выбора стратегии управления для каждого из пяти рисков без требования детального плана реагирования;
6. краткого вывода о наиболее опасных рисках проекта.

---

## Расширенная версия задания

Для сильной группы можно добавить:

1. проведение экспертного опроса для оценки вероятности и влияния с агрегацией мнений (метод Дельфи);
2. расчёт кумулятивного RPN для каждой категории рисков и сравнительный анализ категорий;
3. построение диаграммы Парето для визуализации вклада TOP-10 рисков в общий уровень рискованности проекта;
4. разработку полного плана реагирования для каждого риска TOP-10 с бюджетной оценкой каждой меры;
5. анализ чувствительности — пересчёт RPN при изменении оценок вероятности и влияния на ±1 балл;
6. сравнение полученного реестра с типовыми рисками для аналогичных ИИ-проектов (на основе литературных данных или открытых источников);
7. разработку рекомендаций по интеграции риск-менеджмента в процесс управления проектом на всех стадиях жизненного цикла.
