# КИМ 3. Практическая работа "Формирование AI-команды проекта"
# Варианты индивидуализации задания

## Вариант A. Базовый

Кейс — разработка модели прогнозирования оттока клиентов. Спроектировать структуру команды, построить RACI-матрицу для 4 этапов, описать один конфликт интересов.

## Вариант B. Распределённая команда

Спроектировать команду и RACI-матрицу для сценария, где часть команды работает удалённо в разных часовых поясах. Описать дополнительные риски коммуникации.

## Вариант C. Масштабирование от MVP к продакшн

Сравнить состав команды и матрицу RACI на этапе MVP и на этапе промышленного внедрения AI-продукта.

## Вариант D. Собственный кейс

Выполнить задание на собственном описанном AI-проекте (реальном или учебном). Обязательно обосновать выбор ролей и этапов для RACI-матрицы.

## Вариант E. Метрики эффективности команды

Предложить и обосновать набор метрик для оценки эффективности R&D-команды (например, скорость поставки, качество коммуникации, число конфликтов и т.д.).

## Вариант F. Конфликт заказчик–команда

Углублённо разобрать конфликт интересов между бизнес-заказчиком и командой разработки по критериям приёмки AI-продукта; предложить процедуру согласования критериев качества.
