# КИМ 3. Практическая работа "Формирование AI-команды проекта"
# Методические указания для преподавателя

## Назначение

Практическая работа предшествует формированию AI-проектов: до распределения работ и AI-ролей участники должны пройти диагностику ролевых предпочтений и получить обоснованное распределение функций внутри AI-команды.

## Рекомендации по проведению

1. Работа выполняется в полном составе AI-команды за одно занятие; тестирование по 4 методикам занимает наибольшую часть времени.
2. Рекомендуется заранее разбить участников на постоянные AI-команды по 5 человек, чтобы итоговая матрица ролей была применима к дальнейшей работе над AI-проектом.
3. Обработку результатов по каждой методике (перевод ответов в баллы по ключу) целесообразно проверять выборочно, так как ошибки в подсчёте искажают весь дальнейший анализ.
4. Построение гистограмм по AI-команде лучше проводить совместно, чтобы участники увидели общую картину распределения ролей до формирования итоговой матрицы.
5. Разрешение конфликтов ролей (Шаг 5) рекомендуется проводить как модерируемое обсуждение, фиксируя аргументы каждой стороны.

## Что проверять в первую очередь

- пройдены ли все 4 методики каждым участником AI-команды;
- соблюдено ли правило распределения баллов внутри разделов тестов (сумма = 10 для Белбина и Немова);
- корректно ли применён ключ обработки для каждой методики;
- построены ли гистограммы и сделан ли по ним содержательный вывод;
- учитывают ли индивидуальные ранжированные профили результаты всех 4 методик, а не одной-двух;
- обеспечена ли функциональная полнота AI-команды (все 5 функциональных блоков);
- аргументировано ли разрешение конфликтов ролей числовыми данными, а не только предпочтениями участников.

## Типичные ошибки студентов

1. Нарушение правила распределения баллов в разделах тестов Белбина и Немова (сумма не равна 10).
2. Неверное применение ключа при переносе ответов в таблицу подсчёта.
3. Индивидуальный ранжированный профиль строится только по одной методике (обычно по Белбину), без учёта DISC, Немова и Деллингера.
4. Итоговая матрица распределения ролей формируется без учёта функциональной полноты AI-команды — например, отсутствует роль, отвечающая за анализ и контроль.
5. Конфликт ролей разрешается волевым решением без фиксации в отчёте, какие числовые данные учитывались.
6. Фигура №5 по Деллингеру или роль с наименьшим баллом по Белбину ошибочно назначается как основная роль участника.
7. Гистограммы строятся, но не анализируются (не указаны перегруженные и дефицитные роли).

## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. прохождения двух методик (Белбин и DISC);
2. построения гистограмм по этим методикам;
3. упрощённого ранжированного профиля (без разделения на зону развития);
4. итоговой матрицы распределения ролей без глубокого обоснования конфликтов.

## Расширенная версия задания

Для сильной группы можно добавить:

1. количественную оценку степени сбалансированности AI-команды (например, индекс покрытия функциональных блоков);
2. анализ устойчивости распределения ролей при изменении состава AI-команды (что произойдёт, если один участник выбудет);
3. сравнение результатов методик между собой и обсуждение случаев противоречий (например, лидер по Белбину, но интроверт по DISC);
4. рекомендации по развитию участников в их «зонах развития/стресса».
