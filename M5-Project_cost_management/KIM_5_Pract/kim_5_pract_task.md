# КИМ 5.Практическая работа "Управление стоимостью проекта"

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Модуль 5. Управление стоимостью проекта |

| Вид контроля | Текущий контроль |

## 2. Назначение КИМ

КИМ предназначен для проверки того, что обучающийся понимает ключевые технико-экономические показатели внедрения ИИ, умеет строить расчётные модели для оценки этих показателей и применять методику освоенного объёма (Earned Value Management, EVM) для контроля бюджета и прогнозирования результатов выполнения AI-проекта.

Задание также проверяет понимание специфики бюджетирования и контроля стоимости в ИИ-проектах: неравномерность затрат на эксперименты, вычислительные ресурсы, разметку данных и т.д.

## 3. Проверяемый результат обучения

Обучающийся знает ключевые технико-экономические показатели внедрения ИИ; строит расчётные модели для оценки технико-экономических показателей ИИ и применяет их для прогнозирования.

## 4. Проверяемые результаты и компетенции

| Компетенция / область результата | Ориентировочный индикатор | Уровень | Как проявляется в задании |
|---|---|---|---|
| LC-7 | 7.1 | Базовый/Средний | Расчёт бюджета AI-проекта, применение методики освоенного объёма (EVM), прогнозирование результатов выполнения проекта. |

## 5. Исходные данные (кейс проекта)

Для выполнения задания используется учебный кейс AI-проекта (может совпадать с кейсом из предыдущих модулей), содержащий:

1. плановый бюджет проекта (Budget At Completion, BAC) с разбивкой по этапам/статьям затрат (данные, вычислительные ресурсы, оплата труда команды, инфраструктура и т.д.);
2. плановый календарный график освоения бюджета;
3. фактические данные о выполнении и затратах на определённую контрольную дату (могут быть заданы преподавателем или смоделированы обучающимся при обосновании).

## 6. Программные средства

Рекомендуемый стек:

- MS Excel / Google Sheets — для расчёта показателей EVM и построения S-кривых;
- при необходимости — Python (`pandas`, `matplotlib`) для автоматизации расчётов и визуализации.

## 7. Ход выполнения

### Часть 1. Формирование бюджета проекта

1. Постройте таблицу плановой стоимости работ (Planned Value, PV) по этапам проекта на основе календарного плана (при наличии — используйте план из модуля 4).
2. Определите общий бюджет проекта по завершении (Budget At Completion, BAC).

### Часть 2. Расчёт показателей освоенного объёма (EVM)

На заданную контрольную дату рассчитайте:

- Planned Value (PV) — плановая стоимость запланированных работ;
- Earned Value (EV) — освоенный объём (плановая стоимость фактически выполненных работ);
- Actual Cost (AC) — фактическая стоимость выполненных работ;
- Cost Variance (CV = EV − AC) и Schedule Variance (SV = EV − PV);
- Cost Performance Index (CPI = EV / AC) и Schedule Performance Index (SPI = EV / PV).

### Часть 3. Прогнозирование результатов проекта

1. Рассчитайте прогнозную стоимость проекта по завершении (Estimate At Completion, EAC) минимум двумя способами (например, `EAC = BAC / CPI` и `EAC = AC + (BAC − EV)`).
2. Рассчитайте прогноз стоимости оставшихся работ (Estimate To Complete, ETC) и отклонение по завершении (Variance At Completion, VAC).
3. Интерпретируйте полученные показатели: опережает ли проект бюджет и график, или отстаёт.

### Часть 4. Специфика бюджетирования AI-проектов

Опишите минимум два фактора, специфичных для бюджетирования и контроля стоимости в ИИ-проектах, например:

- неравномерные и трудно прогнозируемые затраты на эксперименты по подбору гиперпараметров и обучению моделей;
- затраты на вычислительные ресурсы (облачные GPU/TPU), зависящие от числа и длительности экспериментов;
- затраты на разметку и подготовку данных, которые могут расти при обнаружении проблем с качеством данных;
- риск незапланированных итераций из-за недостижения целевых метрик качества модели.

### Часть 5. Визуализация

Постройте график освоенного объёма (S-кривые PV, EV, AC) по этапам/периодам проекта и отметьте на нём контрольную дату.

### Часть 6. Дополнительное исследование

Выполните минимум одно дополнительное исследование:

- смоделируйте сценарий дополнительной итерации обучения модели (например, из-за недостижения целевой метрики) и пересчитайте EAC/VAC;
- сравните два способа расчёта EAC и объясните, когда какой предпочтительнее;
- предложите способ бюджетирования резерва на риски, специфичные для ИИ-проектов (management reserve / contingency reserve);
- проанализируйте чувствительность CPI/SPI к изменению одного из этапов проекта.

## 8. Формат сдачи

Обучающийся сдаёт:

1. таблицу расчёта PV, EV, AC, CV, SV, CPI, SPI на контрольную дату;
2. расчёт EAC, ETC, VAC минимум двумя способами;
3. график S-кривых (PV, EV, AC);
4. аналитическую записку с интерпретацией показателей и обсуждением специфики бюджетирования AI-проектов;
5. (при выполнении) материалы дополнительного исследования.

## 9. Требования к отчёту

Отчёт должен содержать:

1. описание кейса AI-проекта и структуры бюджета;
2. таблицу показателей EVM на контрольную дату;
3. расчёт и интерпретацию EAC, ETC, VAC;
4. график S-кривых;
5. анализ специфических факторов стоимости ИИ-проектов;
6. вывод о текущем статусе проекта (опережение/отставание по бюджету и графику) и рекомендации.

## 10. Контрольные вопросы

1. Что такое Planned Value, Earned Value и Actual Cost и как они соотносятся друг с другом?
2. Как рассчитываются Cost Variance и Schedule Variance и что означает их положительное/отрицательное значение?
3. Что показывают индексы CPI и SPI и как их интерпретировать?
4. Какими способами можно рассчитать Estimate At Completion (EAC) и в чём разница между ними?
5. Что такое Variance At Completion (VAC) и о чём он говорит?
6. Какие факторы делают бюджетирование ИИ-проектов более сложным по сравнению с классическими проектами?
7. Как затраты на вычислительные ресурсы (облачные GPU/TPU) влияют на структуру бюджета AI-проекта?
8. Зачем в бюджете AI-проекта закладывать резерв на риски, связанные с экспериментами и качеством данных?
9. Как S-кривые помогают визуально контролировать освоение бюджета проекта?
10. Какие управленческие решения может потребовать низкое значение CPI при удовлетворительном SPI (и наоборот)?

## 11. Использование ИИ-инструментов

Разрешается использовать генеративные ИИ-инструменты для:

- пояснения формул и логики методики освоенного объёма;
- генерации шаблона таблицы расчёта;
- проверки текста аналитической записки.

Обязательно указывается:

1. какой инструмент использовался;
2. для какой части работы;
3. какие фрагменты были изменены после проверки;
4. как обучающийся убедился в корректности расчётов (например, путём проверки формул вручную).

Не допускается сдача расчётов EVM, полностью выполненных ИИ без самостоятельной проверки корректности формул и итоговых значений.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

Итоговый вклад КИМ в БРС рассчитывается по формуле:

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```

Такой подход позволяет использовать единую рубрику для всех КИМ, но не фиксировать внутри отдельного задания общий вес текущего контроля, проекта и экзамена.
