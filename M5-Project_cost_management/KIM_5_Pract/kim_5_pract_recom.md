# КИМ 5. Практическая работа "Управление стоимостью проекта"
# Методические указания для преподавателя

## Назначение

Этот КИМ используется как образец оформления модульного практического задания в ФОС. Его можно применять как текущий контроль по модулю 5 или как часть комплексного КИМ, связанного с экономикой AI-проекта.

## Рекомендации по проведению

1. До занятия рекомендуется напомнить студентам базовые понятия EVM (PV, EV, AC) на простом числовом примере без AI-специфики.
2. На занятии целесообразно сначала разобрать расчёт CPI/SPI и их интерпретацию, а затем перейти к прогнозным показателям (EAC, ETC, VAC).
3. Формирование бюджета AI-кейса и расчёт базовых показателей можно выполнить под руководством преподавателя.
4. Анализ специфики бюджетирования ИИ-проектов и дополнительное исследование рекомендуется выполнять самостоятельно.

## Что проверять в первую очередь

- корректно ли рассчитан Earned Value (частая ошибка — подмена EV фактическими затратами);
- верно ли интерпретированы CPI и SPI (отставание/опережение по стоимости и графику);
- рассчитаны ли прогнозные показатели минимум двумя способами;
- отражают ли S-кривые контрольную дату и реальную динамику проекта;
- содержательно ли раскрыта специфика бюджетирования именно ИИ-проектов (эксперименты, вычислительные ресурсы, разметка данных), а не общие проектные риски.

## Типичные ошибки студентов

1. Путают Earned Value с фактическими затратами (Actual Cost).
2. Не интерпретируют полученные значения CPI/SPI управленчески (просто приводят числа).
3. Используют только один способ расчёта EAC без обоснования.
4. Строят S-кривые без привязки к конкретной контрольной дате.
5. Подменяют специфику ИИ-проектов общими рисками управления стоимостью, характерными для любых проектов.

## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. расчёта PV, EV, AC на одну контрольную дату;
2. расчёта CV, SV, CPI, SPI;
3. расчёта EAC одним способом;
4. краткого описания одного фактора специфики ИИ-бюджетирования.

## Расширенная версия задания

Для сильной группы можно добавить:

1. расчёт EAC двумя-тремя способами со сравнением;
2. сценарный анализ с дополнительной итерацией обучения модели;
3. анализ чувствительности CPI/SPI к изменению одного из этапов;
4. проектирование резерва на риски (management reserve / contingency reserve), специфичные для ИИ-проектов.
