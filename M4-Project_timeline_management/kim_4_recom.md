# Методические указания для преподавателя

## Назначение

Этот КИМ используется как образец оформления модульного практического задания в ФОС. Его можно применять как текущий контроль по модулю 4 или как часть комплексного КИМ по планированию AI-проекта.

## Рекомендации по проведению

1. До занятия рекомендуется напомнить студентам базовые понятия сетевого планирования (метод критического пути, диаграмма Ганта) из общего курса управления проектами.
2. На занятии целесообразно сначала разобрать расчёт критического пути на простом учебном примере без AI-специфики.
3. Построение сетевой модели и диаграммы Ганта для AI-кейса можно выполнить под руководством преподавателя.
4. Анализ неопределённости и корректировку плана рекомендуется выполнять самостоятельно.

## Что проверять в первую очередь

- корректно ли определены зависимости между операциями;
- верно ли рассчитан критический путь и резервы времени;
- отражает ли диаграмма Ганта реальные зависимости задач;
- содержательно ли описаны источники неопределённости именно для AI-проекта (а не общие проектные риски);
- корректно ли пересчитан план при смоделированном изменении условий.

## Типичные ошибки студентов

1. Указывают неполный или некорректный список зависимостей между операциями.
2. Ошибаются в расчёте ранних/поздних сроков и резервов времени.
3. Строят диаграмму Ганта без учёта зависимостей задач.
4. Ограничиваются общими проектными рисками, не привязывая неопределённость к специфике AI (эксперименты, данные, метрики моделей).
5. Не пересчитывают критический путь после внесения изменений в план.

## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. таблицы операций с зависимостями (5–7 операций);
2. расчёта критического пути вручную;
3. построения диаграммы Ганта;
4. описания одного источника неопределённости.

## Расширенная версия задания

Для сильной группы можно добавить:

1. оптимизацию «время − затраты» (crashing);
2. буферную стратегию по методу критической цепи;
3. сравнение классического и итеративного планирования;
4. сравнение влияния CRISP-DM и CRISP-ML(Q) на структуру плана.
