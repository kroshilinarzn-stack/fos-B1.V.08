# КИМ 4. Лабораторная работа "Разработка сетевой модели проекта"

# Методические указания для преподавателя

## Назначение

Работа продолжает цикл лабораторных работ по управлению AI-проектом: после формирования AI-команды и перечня работ AI-проекта обучающиеся переходят к сетевому планированию сроков — построению сетевого графика, расчёту критического пути и диаграммы Ганта.

## Рекомендации по проведению

1. До занятия рекомендуется убедиться, что у каждого обучающегося (или AI-команды) есть готовый сводный перечень работ с зависимостями из предыдущей лабораторной работы.
2. На занятии целесообразно сначала разобрать вручную небольшой пример (5–7 работ), чтобы студенты усвоили порядок прямого и обратного прохода по сети, прежде чем переходить к полному перечню работ.
3. Построение сетевого графика и первичную проверку на циклы рекомендуется выполнить под контролем преподавателя.
4. Расчёт параметров (ES/EF/LS/LF, резервы) можно оставить самостоятельным, но с промежуточной проверкой минимум одного пути вручную.
5. Анализ загрузки ресурсов и оптимизацию сетевого графика лучше вынести в отчёт как отдельный содержательный блок, обсуждаемый на защите.

## Что проверять в первую очередь

- содержит ли перечень работ зависимости для всех работ, кроме исходных;
- нет ли циклов и изолированных работ в построенном сетевом графике;
- корректно ли выполнен прямой проход по сети (ES/EF);
- корректно ли выполнен обратный проход по сети (LS/LF);
- совпадает ли для критических работ полный резерв с нулём;
- обоснован ли выбор критического пути ссылкой на конкретные значения резервов;
- отражает ли диаграмма Ганта критический путь и резервы некритических работ;
- есть ли содержательный анализ загрузки ресурсов, а не формальное упоминание.

## Типичные ошибки студентов

1. В перечне работ отсутствуют зависимости для части работ, из-за чего сетевой график получается несвязным.
2. При прямом проходе по сети не учитываются все предшественники работы (берётся только один вместо максимума по всем).
3. При обратном проходе по сети не учитываются все последующие работы (берётся не минимум, а произвольное значение).
4. Критический путь определяется «на глаз» по диаграмме Ганта без проверки нулевого резерва.
5. Путают полный и свободный резерв времени, используют их как взаимозаменяемые.
6. Диаграмма Ганта строится по длительностям без учёта рассчитанных ранних сроков начала работ.
7. Анализ загрузки ресурсов ограничивается фразой «ресурсы распределены равномерно» без количественного обоснования.
8. Расчёты представлены без исходной таблицы работ, что делает их невоспроизводимыми.

## Минимальная версия задания

Для слабой группы можно сократить задание до:

1. перечня из 10–12 работ с явными зависимостями;
2. построения сетевого графика в нотации «работы-вершины»;
3. расчёта только ES/EF и определения ориентировочной длительности проекта;
4. построения диаграммы Ганта без выделения резервов;
5. краткого вывода о критических работах.

## Расширенная версия задания

Для сильной группы можно добавить:

1. полный расчёт LS/LF, TF и FF для всех работ;
2. сравнение двух нотаций построения сетевого графика на одном перечне работ;
3. краш-анализ — количественную оценку стоимости сокращения критического пути;
4. анализ нескольких параллельных критических путей (при их наличии);
5. моделирование сценария задержки конкретной работы и пересчёт всего графика.
