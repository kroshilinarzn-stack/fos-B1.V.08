**2. НАЗНАЧЕНИЕ КИМ**

КИМ предназначен для проверки того, что обучающийся умеет:

- применять инструменты стратегического планирования (Impact Mapping)
  для связывания бизнес-целей с действиями команды;

- переводить стратегические цели в пользовательские истории и
  функциональные требования с помощью User Story Mapping;

- структурировать содержание проекта с помощью WBS (Work Breakdown
  Structure);

- фиксировать основные параметры проекта в формализованном документе ---
  Уставе проекта;

- определять чёткие границы и критерии успеха проекта на ранних этапах
  планирования.

Задание проверяет способность студента пройти путь от **бизнес-цели до
структурированного плана работ**, используя современные инструменты
продуктового и проектного менеджмента в контексте AI-проектов.

**3. ПРОВЕРЯЕМЫЙ РЕЗУЛЬТАТ ОБУЧЕНИЯ**

Обучающийся разрабатывает Устав AI-проекта, применяет методики Impact
Mapping, User Story Mapping и WBS для формализации содержания, границ и
критериев успеха проекта, а также обосновывает выбранные решения с
учётом специфики ИИ-решений и потребностей бизнеса.

**5. ИСХОДНЫЕ ДАННЫЕ (ВЫБОР ТЕМАТИКИ ПРОЕКТА)**

Тематика проекта определяется преподавателем или выбирается студентом из
предложенного пула:

1.  **Разработка AI-ассистента** для call-центра сети телеком-оператора
    (повышение скорости ответа).

2.  **Система генеративной рекомендации** товаров для интернет-магазина
    стройматериалов (увеличение среднего чека).

3.  **Предиктивная аналитика** для управления цепочками поставок в
    логистической компании (снижение задержек).

4.  **Чат-бот для внутренней поддержки** сотрудников производственного
    предприятия (поиск по базе знаний).

5.  **Автоматическое распознавание и классификация** документов для
    страховой компании (ускорение обработки заявлений).

**Требования к тематике проекта:**\
Тематика проекта обязательно должна содержать:

- Краткое описание организации / домена (например, «B2B-платформа для
  автоматизации юридических услуг», «Сеть кофеен», «Стартап в области
  EdTech»).

- Суть инициативы (например, «внедрение системы генеративной
  рекомендации контента», «разработка AI-ассистента для поддержки
  операторов колл-центра», «создание чат-бота для поиска по внутренней
  документации»).

- Контекст внедрения (пилот / масштабирование, бюджет, сроки,
  технологический стек).

- Бизнес-цель проекта (например: «повысить конверсию клиентов на 10%»,
  «снизить время ответа поддержки на 30%»).

**Важно:** Проект должен быть достаточно сложным, чтобы включать
несколько функциональных областей и потребность в декомпозиции задач.

**6. ПРОГРАММНЫЕ СРЕДСТВА**

Рекомендуемые инструменты:

**7. ХОД ВЫПОЛНЕНИЯ**

**Часть 1. Инициирование проекта: разработка Устава**

1.  Сформулируйте **название проекта** и **краткое описание**.

2.  Опишите **бизнес-обоснование** и **стратегическую цель** проекта
    (SMART-цель).

3.  Определите **критерии успеха** и ключевые **показатели эффективности
    (KPI)** для оценки результата.

4.  Укажите **границы проекта** (что входит и что НЕ входит в
    содержание), ключевые **ограничения** и **допущения**.

5.  Заполните таблицу с **основными участниками проекта** (заказчик,
    спонсор, команда, ключевые стейкхолдеры) и их зонами
    ответственности.

6.  Опишите **масштаб** проекта (пилот, MVP, полномасштабное внедрение).

**Часть 2. Стратегическое планирование: Impact Mapping**

1.  Постройте **Impact Map** для вашего проекта, последовательно
    определив:

    - **Цель (Goal):** Какой бизнес-результат мы хотим достичь?

    - **Акторы (Actors):** Кто может повлиять на результат или кто
      является пользователями решения?

    - **Воздействия (Impacts):** Как акторы должны изменить своё
      поведение, чтобы мы достигли цели?

    - **Результаты (Deliverables):** Что мы должны сделать
      (разработать), чтобы вызвать эти воздействия?

2.  Сформулируйте **гипотезы** для каждого воздействия. Опишите, как вы
    планируете проверять эти гипотезы.

**Часть 3. Продуктовое планирование: User Story Mapping**

1.  На основе Impact Map сформируйте **User Story Map**, включающую:

    - **Шаги пользователя (User Activities/Tasks):** Крупные задачи
      пользователя при работе с системой.

    - **Пользовательские истории (User Stories):** Конкретные истории
      (формат: «Как \[роль\], я хочу \[действие\], чтобы \[ценность\]»).

    - **Приоритетность:** Разделите истории на релизы или итерации
      (например, MVP → Версия 1.0 → Версия 2.0).

2.  Выделите функциональность для первого релиза (MVP) и последующих,
    объяснив приоритизацию с точки зрения ценности для
    бизнеса/пользователя.

3.  Заполните бэклог продукта (список историй).

**Часть 4. Декомпозиция работ: WBS**

1.  На основе User Story Map и общей логики проекта создайте **WBS (Work
    Breakdown Structure)** --- иерархический список всех работ.

2.  WBS должна содержать как минимум **3 уровня декомпозиции**.

3.  Каждому пакету работ присвойте уникальный идентификатор (код WBS).

4.  Для каждого пакета работ нижнего уровня определите ответственного
    (условно) и приблизительную трудоёмкость (человеко-часы или
    сторипоинты).

5.  Свяжите элементы WBS с соответствующими пользовательскими историями
    из части 3.

**Часть 5. Анализ и обоснование**

1.  Опишите взаимосвязь между созданными артефактами:

    - Как цель из устава трансформировалась в Impact Map?

    - Как Impact Map влияет на формирование пользовательских историй?

    - Как User Story Map преобразуется в WBS?

2.  Выделите **«тёмные зоны»** (зоны высокой неопределённости) в
    содержании проекта. Опишите, как вы планируете управлять этой
    неопределённостью (например, спринты для исследования гипотез,
    прототипирование).

3.  Сделайте выводы о том, как использованные инструменты помогают
    определить границы проекта.

**9. ФОРМАТ СДАЧИ**

Обучающийся сдаёт единый отчёт в электронном виде, содержащий:

1.  Титульный лист с названием проекта, ФИО, группой.

2.  Основную часть, включающую все таблицы, карты, диаграммы и
    аналитику.

3.  Графическое представление артефактов (скриншоты с Miro, Draw.io и др.).

4.  Выводы по использованным методикам.

**Объём аналитической записки:** 4--6 страниц (без учёта приложений и
визуализаций).

**10. ТРЕБОВАНИЯ К ОТЧЁТУ**

1.  **Устав проекта** полностью заполнен и соответствует
    SMART-критериям.

2.  **Impact Map** логически выводит цели в конкретные результаты,
    содержит не менее 3-х акторов и 2-х уровней воздействия.

3.  **User Story Map** содержит не менее 15--20 пользовательских
    историй, разбитых как минимум на 3 релиза.

4.  **WBS** содержит не менее 20--25 пакетов работ, структурирована
    логически, имеет не менее 3 уровней и корректные ID.

5.  Сделаны выводы о роли каждого из артефактов в управлении содержанием
    AI-проекта.

**12. ИСПОЛЬЗОВАНИЕ ИИ-ИНСТРУМЕНТОВ**

Разрешается использовать генеративные ИИ-инструменты для:

- Генерации первичных идей для Impact Mapping и User Stories на основе
  описания проекта.

- Предложения структуры WBS для различных типов AI-проектов.

- Помощи в формулировке пользовательских историй в правильном формате.

- Проверки корректности и полноты Устава проекта.

**Обязательно указывается:**

1.  Какой инструмент использовался.

2.  Для какой части работы.

3.  Какие фрагменты были изменены после проверки человеком.

4.  Как обучающийся оценил применимость сгенерированных результатов к
    своему конкретному кейсу.

**Не допускается:**

- Сдача полностью сгенерированной ИИ работы без самостоятельного анализа
  и доработки.

- Использование шаблонов, не адаптированных под тематику выбранного
  проекта.

**13. Пересчёт в БРС**

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в
итоговой балльно-рейтинговой системе определяется в
документе docs/evaluation_policy.md.

Итоговый вклад КИМ в БРС рассчитывается по формуле:\
**балл_БРС = (балл_КИМ / 10) × вес_КИМ**
