# КИМ 2. Тест по теме "Управление содержанием AI-проекта"
## Варианты индивидуализации задания

### Вариант 1: Устное собеседование по изученным темам

**Формат:** Диалог преподавателя со студентом в формате «вопрос — ответ».

**Вопросы (кейсы) для обсуждения:**

1.  Представьте, что вы менеджер AI-проекта. Бизнес-спонсор поставил цель: «Повысить точность прогнозирования спроса на 20%». Как вы, используя инструмент Impact Mapping, разложите эту цель до конкретных действий команды разработки? Опишите пошаговый процесс.

2.  В чем принципиальное отличие User Story Mapping от традиционного бэклога требований (например, в Waterfall)? Почему для AI-проектов этот инструмент особенно полезен?

3.  Опишите процесс создания иерархической структуры работ (WBS) для проекта по разработке системы распознавания лиц для системы безопасности. Какие уровни декомпозиции вы выделите? Приведите пример пакета работ нижнего уровня.

4.  Вы проводите SWOT-анализ для стартапа, который разрабатывает AI-решение для диагностики заболеваний по медицинским изображениям. Какие факторы вы отнесете к «Возможностям», а какие — к «Угрозам»? Обоснуйте свой выбор.

5.  Какие этапы включает цикл DMAIC в методологии «Шесть сигм»? Опишите, как этот цикл может быть применен для улучшения процесса сбора и разметки данных в AI-проекте.

6.  Разработайте структуру Устава для проекта по внедрению рекомендательной системы в интернет-магазине. Какие разделы вы включите и почему они важны для авторизации AI-проекта?

7.  Как бы вы проанализировали и классифицировали стейкхолдеров проекта по внедрению AI-системы для автоматического распознавания дефектов на производственной линии? Какие стратегии взаимодействия с каждой группой вы предложите?

8.  Опишите логическую последовательность применения инструментов Impact Mapping, User Story Mapping и WBS при планировании содержания AI-проекта. Как каждый из этих инструментов дополняет предыдущий и подготавливает данные для следующего?

9.  В чем заключается специфика управления содержанием AI-проектов по сравнению с проектами по разработке традиционного ПО? Какие дополнительные факторы (связанные с данными, моделями, этичностью) необходимо учитывать при определении границ проекта?

10. Вы − руководитель проекта по созданию AI-сервиса для прогнозирования оттока клиентов. На этапе планирования выясняется, что доступных исторических данных недостаточно для достижения целевых бизнес-показателей. Как вы будете действовать? Какие документы и процессы управления содержанием проекта (Устав, WBS, User Story Map) вы задействуете для корректировки плана?

### Вариант 2: Разработка карты пользовательских историй (User Story Map) для AI-функционала

**Формат:** Инфографика (схема User Story Map) + пояснительная записка.

**Описание:** Разработайте User Story Map для одного из сценариев использования AI-функционала (на выбор):

- Сценарий А: Пользователь загружает изображение в AI-систему для автоматической классификации дефектов.
- Сценарий Б: Пользователь вводит текстовый запрос в AI-помощника для получения прогнозной аналитики по продажам.
- Сценарий В: Пользователь настраивает параметры AI-модели через веб-интерфейс администратора.

**Карта должна включать:**

1.  **«Скелет» (Backbone):** последовательность действий пользователя (активности и шаги). **(LC-1.1)**
2.  **Пользовательские истории:** детализацию требований для каждого шага. **(LC-1.1)**
3.  **Выделение MVP:** четкое обозначение историй, входящих в первый релиз (минимально жизнеспособный продукт). **(LC-1.1)**
4.  **Пояснительную записку** с обоснованием выбора MVP и описанием того, как эта карта помогает управлять содержанием проекта и коммуницировать требования к AI-системе. **(LC-1.3)**

**Критерии оценивания:**

| Полнота и корректность построения «скелета» (Backbone) карты | 3 |
| Детализация пользовательских историй (учет специфики AI-функционала) | 3 |
| Корректность выделения MVP и его обоснование | 2 |
| Качество оформления и пояснительной записки | 2 |
| **Итого** | **10** |

### Вариант 3: Сравнительный анализ применения инструментов управления содержанием в двух AI-проектах

**Формат:** Сравнительная таблица + аналитическая записка (4-6 страниц).

**Описание:** Выберите два гипотетических AI-проекта, различающихся по сложности и масштабу (например, «Чат-бот для технической поддержки» и «Система прогнозирования кибератак с использованием глубокого обучения»). Проведите сравнительный анализ применения инструментов управления содержанием:

1.  Применимость и адаптация **Impact Mapping** для каждого проекта: как связать бизнес-цели с поведением стейкхолдеров. **(LC-1.1)**
2.  Применимость и адаптация **User Story Mapping** для каждого проекта: как структурировать пользовательский опыт и выделить MVP с учетом специфики AI. **(LC-1.1)**
3.  Применимость и адаптация **WBS** для каждого проекта: как выполнить декомпозицию работ, связанных с данными и моделями. **(LC-1.1)**
4.  Состав проектной документации, которая потребуется для каждого проекта, с обоснованием необходимости ее адаптации под AI-проект. **(LC-1.3)**

### Вариант 4: Инфографика: «Дорожная карта планирования содержания AI-проекта»

**Формат:** Инфографика (постер, схема) + краткое описание (1-2 страницы).

**Описание:** Визуализируйте процесс планирования содержания AI-проекта от бизнес-цели до детального плана работ, включив:

1.  **Блок-схему** последовательного применения инструментов: MBO (SMART-цели) → Impact Mapping → User Story Mapping → WBS. **(LC-1.1)**
2.  **Перечень артефактов (документов)**, создаваемых на каждом этапе планирования: Устав проекта, Описание содержания проекта, Словарь WBS, User Stories и т.д. **(LC-1.3)**
3.  **Визуальное отображение взаимосвязей:** покажите, как результаты одного инструмента становятся входными данными для следующего. **(LC-1.1)**
4.  **Акценты на специфике AI-проектов:** выделите на схеме те этапы планирования, где учитывается работа с данными, неопределенность моделей, этические аспекты. **(LC-1.2)**

### Вариант 5: Разработка Устава AI-проекта на основе бизнес-кейса

**Формат:** Готовый структурированный документ (Устав проекта) + презентация.

**Описание:** На основе предложенного преподавателем бизнес-кейса (например, «Внедрение AI-системы для прогнозирования оттока клиентов в телеком-компании») разработайте Устав проекта, который включает:

1.  **Бизнес-цели и обоснование** с использованием SMART-критериев. **(LC-1.1)**
2.  **Высокоуровневое описание содержания** и ключевых результатов (что создается, а что не входит в проект). **(LC-1.1)**
3.  **Предварительные сроки и вехи** с привязкой к контрольным точкам. **(LC-1.1)**
4.  **Анализ ключевых стейкхолдеров** (матрица "Влияние/Интерес") с определением стратегий взаимодействия. **(LC-1.1)**
5.  **Предварительный реестр рисков**, специфичных для AI-проекта (данные, модели, этика). **(LC-1.2)**
6.  **Презентацию** (5-7 слайдов) для защиты Устава перед спонсором проекта. **(LC-1.3)**
