# Рубрика оценивания теста по теме "Управление содержанием AI-проекта"

## Оценивание КИМ_2_Test

Каждый вопрос оценивается в 1 балл, максимальный балл за тест — **10 баллов**.

### Шкала оценивания

| Оценка по 5-балльной шкале | Процент правильных ответов | Количество баллов (из 10) |
| :--- | :--- | :--- |
| **«Отлично»** | 90–100% | 9–10 баллов |
| **«Хорошо»** | 70–89% | 7–8 баллов |
| **«Удовлетворительно»** | 50–69% | 5–6 баллов |
| **«Неудовлетворительно»** | 0–49% | 0–4 балла |

### Рекомендации по интерпретации результатов

| Результат | Интерпретация |
| :--- | :--- |
| **Высокий балл (9–10 баллов)** | Обучающийся демонстрирует системное понимание инструментов управления содержанием проекта (MBO, Impact Mapping, User Story Mapping, WBS, SWOT-анализ, «Шесть сигм»), осознает их взаимосвязь и специфику применения в AI-проектах. Умеет формализовать бизнес-цели, анализировать стейкхолдеров и среду, а также уверенно владеет структурой и содержанием ключевой проектной документации (Устав, WBS). Компетенции **LC-1.1, LC-1.2, LC-1.3** сформированы на **высоком уровне**. |
| **Средний балл (7–8 баллов)** | Обучающийся в целом освоил понятийный аппарат и основные инструменты, но допускает отдельные ошибки в вопросах, требующих анализа нескольких правильных вариантов (множественный выбор) или установления логических связей (сопоставление, упорядочивание). Испытывает незначительные трудности в применении знаний к контексту AI-проектов или при выборе оптимального инструмента для конкретной задачи. Компетенции сформированы на **достаточном уровне**. |
| **Пороговый балл (5–6 баллов)** | Обучающийся усвоил преимущественно фактологический материал (определения и назначение инструментов), но испытывает сложности при выполнении заданий на сопоставление и упорядочивание. Отмечается недостаточное понимание взаимосвязей между инструментами (Impact → USM → WBS) и этапами планирования содержания. Компетенции сформированы на **базовом уровне** и требуют дополнительной проработки. |
| **Низкий балл (0–4 балла)** | Обучающийся не усвоил значительную часть учебного материала по модулю. Демонстрирует фрагментарные знания, не способен применить инструменты планирования в практическом контексте, не понимает специфики AI-проектов и не ориентируется в структуре проектной документации. Компетенции **не сформированы**. Рекомендуется дополнительное изучение теоретического материала и повторное прохождение тестирования. |

### Дополнительные рекомендации по анализу результатов

Для углубленной диагностики рекомендуется анализировать результаты не только по общему баллу, но и по группам вопросов. Это позволит выявить конкретные тематические блоки, вызывающие наибольшие затруднения у студентов:

| Группа вопросов | Тематический блок | Что проверяется |
| :--- | :--- | :--- |
| Группа 1 | Основы определения целей. MBO и SMART | Умение формулировать цели и различать их от стратегий |
| Группа 2 | Инструменты визуализации. CJM и Impact Mapping | Понимание логики Impact Mapping и этапов CJM |
| Группа 3 | User Story Mapping | Знание структуры карты и принципов формирования MVP |
| Группа 4 | WBS | Умение декомпозировать работы и понимание уровней WBS |
| Группа 5 | SWOT-анализ | Понимание категорий и их принадлежности к среде |
| Группа 6 | Шесть сигм | Знание цикла DMAIC и его этапов |
| Группа 7 | Устав AI-проекта | Понимание структуры и назначения разделов Устава |
| Группа 8 | Анализ стейкхолдеров | Умение идентифицировать и анализировать стейкхолдеров |
| Группа 9 | Интеграция инструментов | Понимание логической последовательности применения инструментов |
| Группа 10 | Применение знаний в AI-контексте | Умение адаптировать инструменты к специфике AI-проектов |

Высокий процент ошибок в конкретной группе сигнализирует о необходимости дополнительного разбора соответствующей темы на семинарском занятии или консультации.
