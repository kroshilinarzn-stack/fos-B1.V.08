# КИМ_2_Test. Управление содержанием AI-проекта

## 1. Паспорт КИМ

| Параметр | Значение |
| :--- | :--- |
| **Дисциплина** | Управление проектами в сфере ИИ |
| **Модуль** | Модуль 2. Управление содержанием проекта |

| **Вид контроля** | Текущий контроль |
| **Рекомендуемый формат выполнения** | Индивидуально |
| **Рекомендуемая среда** | Любая тестирующая система или система управления обучением (например, LMS Moodle) |

| **Рекомендуемая трудоёмкость** | 4 академических часа (лекции) |

## 2. Назначение КИМ

КИМ предназначен для проверки теоретических знаний и практических навыков обучающихся по ключевым инструментам и методам управления содержанием проекта, включая формализацию бизнес-целей, применение современных подходов к визуализации требований и структурированию работ. Особое внимание уделяется специфике проектов в сфере искусственного интеллекта, а также компетенциям в области разработки проектной документации.

## 3. Проверяемые результаты обучения и компетенции

| Компетенция | Индикатор | Уровень | Как проявляется в задании |
| :--- | :--- | :--- | :--- |
| **LC-1** | **1.1** | Средний | Понимание и применение инструментов для перехода от бизнес-целей к стратегии внедрения ИИ (Impact Mapping, MBO, SMART), знание методологии анализа стейкхолдеров. |
| **LC-1** | **1.2** | Базовый | Понимание специфики AI-проектов и влияния технологических факторов на содержание проекта, умение анализировать среду (SWOT). |
| **LC-1** | **1.3** | Высокий | Уверенное владение структурой и содержанием ключевых проектных документов (Устав, WBS), применение инструментов планирования содержания (USM, WBS) для подготовки документации. |

## 5. Структура теста

Банк тестовых вопросов состоит из **10 групп**, каждая из которых соответствует отдельному тематическому блоку модуля:

| № группы | Тематический блок |
| :--- | :--- |
| 1 | Основы определения целей. MBO и SMART |
| 2 | Инструменты визуализации. CJM и Impact Mapping |
| 3 | Управление пользовательскими историями (User Story Mapping) |
| 4 | Структурирование содержания проекта (WBS) |
| 5 | Анализ среды. Техника SWOT-анализа |
| 6 | Методология улучшения процессов. Шесть Сигм |
| 7 | Устав AI-проекта. Назначение и структура |
| 8 | Анализ стейкхолдеров AI-проекта |
| 9 | Интеграция инструментов (Impact, USM, WBS) |
| 10 | Применение знаний в контексте AI-проекта |

### Типология заданий

Каждая группа включает **4 вопроса** различных типов:

| Тип вопроса | Назначение |
| :--- | :--- |
| Одиночный выбор | Проверка знания точных определений и однозначных фактов |
| Множественный выбор | Проверка способности идентифицировать несколько правильных признаков, характеристик, элементов |
| Сопоставление | Проверка умения устанавливать логические связи между понятиями и их определениями, ролями и функциями, типами и характеристиками |
| Упорядочивание | Проверка знания последовательности процессов, этапов, фаз жизненного цикла и шагов применения инструментов |

### Порядок формирования теста

Для каждого студента вопросы выбираются случайным образом — по 1 вопросу из каждой группы (итого **10 вопросов**), что обеспечивает:

- эквивалентность вариантов теста;
- индивидуализацию заданий для каждого испытуемого;
- объективность оценки.

Время выполнения теста устанавливается из расчета **1 минуты на вопрос** (итого **10 минут**).

### Валидность и надежность КИМ

**Валидность** обеспечивается следующими факторами:

- **Содержательная валидность:** содержание теста полностью соответствует тематике лекций Модуля 2 и охватывает все заявленные темы, что подтверждается явной связью вопросов с проверяемыми индикаторами компетенций.
- **Конструктная валидность:** структура теста (10 групп по 4 типа вопросов) позволяет оценить не только знание отдельных определений, но и способность к анализу (множественный выбор), синтезу (сопоставление) и логическому мышлению (упорядочивание), что соответствует комплексному характеру проверяемых компетенций.
- **Критериальная валидность:** все задания сформулированы на основе фактического материала, изложенного в учебных пособиях и лекциях, а критерии оценивания четко определены и стандартизированы, что связывает результаты теста с реальными учебными достижениями.

**Надежность** теста обусловлена:

- стандартизированным форматом вопросов, обеспечивающим единообразие интерпретации;
- использованием случайной выборки вопросов из каждой группы, что формирует множество равнозначных вариантов теста, снижает влияние субъективных факторов и минимизирует возможность списывания;
- четкими критериями оценивания каждого типа заданий, гарантирующими объективность проверки.

## 6. Требования к процедуре тестирования

### Условия проведения:

- тест проводится в системе LMS Moodle (или аналогичной электронной образовательной среде);
- рекомендуется проводить тестирование с использованием Safe Exam Browser (SEB) для обеспечения объективности академических достижений;
- доступ к тесту открывается строго во время, отведенное на занятие.

### Материально-техническое обеспечение

- компьютер / ноутбук с доступом в LMS Moodle;
- стабильное интернет-соединение;
- браузер с поддержкой современных веб-стандартов.

## 7. Процедура оценивания

Каждый вопрос оценивается в 1 балл, максимальный балл за тест — **10 баллов**.

### Шкала оценивания

| Оценка по 5-балльной шкале | Процент правильных ответов | Количество баллов (из 10) |
| :--- | :--- | :--- |
| «Отлично» | 90–100% | 9–10 баллов |
| «Хорошо» | 70–89% | 7–8 баллов |
| «Удовлетворительно» | 50–69% | 5–6 баллов |
| «Неудовлетворительно» | 0–49% | 0–4 балла |

### Рекомендации по интерпретации результатов

| Результат | Интерпретация |
| :--- | :--- |
| **Высокий балл (9–10 баллов)** | Обучающийся демонстрирует системное понимание инструментов управления содержанием проекта (MBO, Impact Mapping, USM, WBS), осознает их взаимосвязь и специфику применения в AI-проектах. Умеет формализовать бизнес-цели, анализировать стейкхолдеров и среду, а также уверенно владеет структурой ключевой проектной документации. Компетенции сформированы на высоком уровне. |
| **Средний балл (7–8 баллов)** | Обучающийся освоил основной понятийный аппарат и инструменты, но допускает отдельные ошибки в вопросах, требующих анализа нескольких правильных вариантов или установления логических связей (сопоставление, упорядочивание). Испытывает незначительные трудности в применении знаний к контексту AI-проектов. Компетенции сформированы на достаточном уровне. |
| **Пороговый балл (5–6 баллов)** | Обучающийся усвоил преимущественно фактологический материал (определения), но испытывает сложности при выполнении заданий на сопоставление и упорядочивание. Отмечается недостаточное понимание взаимосвязей между инструментами и этапами планирования содержания. Компетенции сформированы на базовом уровне и требуют дополнительной проработки. |
| **Низкий балл (0–4 балла)** | Обучающийся не усвоил значительную часть учебного материала по модулю. Демонстрирует фрагментарные знания, не способен применить инструменты планирования в практическом контексте. Компетенции не сформированы. Рекомендуется дополнительное изучение теоретического материала и повторное прохождение тестирования. |
