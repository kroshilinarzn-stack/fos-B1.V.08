# КИМ 8. Коллоквиум "Управление проектами в сфере ИИ (модули 1–8)". Промпт

## Задание

Составь банк вопросов коллоквиума по 2 модулям дисциплины «Управление проектами в сфере ИИ»:
- **Модуль 1. Проекты и среда управления**
- **Модуль 2. Управление содержанием проекта**
- **Модуль 3. Управление человеческими ресурсами проекта**
- **Модуль 4. Управление сроками проекта**
- **Модуль 5. Управление стоимостью проекта**
- **Модуль 6. Управление качеством проекта**
- **Модуль 7. Управление рисками и взаимодействием в проекте**
- **Модуль 8. Управление проектами в сфере искусственного интеллекта: методы, подходы, инструментарий**

Банк должен состоять из **двух отдельных групп вопросов** (теоретические и проблемные), каждая из которых разбита на **5 тематических подгрупп**.

1. **Теоретические вопросы** — по 2–3 вопроса в каждой из 5 подгрупп, в формате «Дайте определение понятия «…»», проверяющие владение ключевой терминологией темы.
2. **Проблемные ситуации (кейсы)** — по 1 вопросу в каждой из 5 подгрупп, описывающие практическую ситуацию из управления AI-проектом, требующую анализа и обоснованного решения с опорой на понятия и инструменты соответствующей темы.

## Требования к формату вопросов (для импорта в Moodle)

- каждый вопрос оформляется как отдельный `<question>` типа `essay` в формате Moodle XML;
- вопросы каждой группы и каждой тематической подгруппы размещаются в отдельном файле `kim_kolokvium_7-8_{group}_gr{N}.xml`, где `{group}` — `theory` или `case`, `N` — номер тематической подгруппы (1–5);
- для теоретических вопросов: `defaultgrade` = 2, `responsefieldlines` = 4, поле `graderinfo` содержит эталонное определение понятия;
- для кейсов: `defaultgrade` = 5, `responsefieldlines` = 15, поле `graderinfo` содержит критерии оценивания ответа (не готовое решение, а ориентиры для проверки);
- текст вопроса оформляется в `questiontext` с `format="html"` и оборачивается в `CDATA`;
- каждый файл начинается с `<?xml version="1.0" encoding="UTF-8"?>` и корневого элемента `<quiz>`.

## Требования к содержанию

- теоретические вопросы формулируются кратко и однозначно, без нескольких трактовок понятия;
- эталонные определения должны быть содержательными (не словарной калькой), пригодными для сверки смысла, а не дословного совпадения;
- кейсы должны быть реалистичными ситуациями, специфичными именно для AI-проектов (риски дрейфа данных, проблемы MLOps, выбор методологии в условиях неопределённости, управление стейкхолдерами в инновационных проектах), а не общими управленческими задачами;
- критерии оценивания кейсов формулируются как 2–4 содержательных пункта, по которым преподаватель сможет быстро оценить полноту и обоснованность ответа.

## Валидация

- каждый сгенерированный XML-файл проверяется на корректность синтаксиса (well-formed XML) перед публикацией;
- количество вопросов в каждой группе и подгруппе сверяется с паспортом КИМ (`kim_7-8_Colloq_task.md`).



## Пример структуры файла для теоретических вопросов

```xml
<?xml version="1.0" encoding="UTF-8"?>
<quiz>
<question type="essay">
    <name>
        <text>Теория. ГрN. Определение понятия</text>
    </name>
    <questiontext format="html">
        <![CDATA[<p>Дайте определение понятию <b>«…»</b>.</p>]]>
    </questiontext>
    <defaultgrade>2</defaultgrade>
    <responsefieldlines>4</responsefieldlines>
    <responseformat>editor</responseformat>
    <graderinfo format="html">
        <![CDATA[<p><b>Эталонный ответ:</b></p>
        <p>…</p>
        <p><b>Критерии оценивания:</b></p>
        <ul>
            <li>2 балла — …</li>
            <li>1 балл — …</li>
            <li>0 баллов — …</li>
        </ul>]]>
    </graderinfo>
</question>
</quiz>
