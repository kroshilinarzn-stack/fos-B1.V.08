# КИМ 8. Коллоквиум "Управление проектами в сфере ИИ (модули 7–8)"

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Охватываемые модули | Модуль 7–Модуль 8 |
| Вид оценочного средства | Коллоквиум (опрос по банку теоретических и проблемных вопросов) |
| Вид контроля | Рубежный контроль |
| Рекомендуемый формат выполнения | Индивидуально |
| Рекомендуемая среда | LMS Moodle (банк вопросов типа «эссе») либо устный опрос с использованием того же банка вопросов |
| Основной артефакт | Письменные (в Moodle) или устные ответы на вопросы двух типов: теоретические и проблемные (кейсы) |
| Рекомендуемая трудоёмкость | 1 академический час |

## 2. Назначение КИМ

КИМ предназначен для комплексной проверки усвоения ключевых тем курса управления AI-проектами: управления рисками, взаимодействия с внешними участниками, выбора гибких методологий и инструментальной среды проектного менеджмента. В отличие от модульных лабораторных, практических работ и тестов, проверяющих отдельные навыки, коллоквиум проверяет как владение терминологией (теоретический блок), так и умение применять её для анализа проблемных ситуаций (блок кейсов), а также целостное понимание курса.

## 4. Источники информации

Для подготовки к коллоквиуму обучающиеся используют материалы лекций, лабораторных и практических работ модулей 7–8, а также источники, указанные в паспортах соответствующих модульных КИМ дисциплины (в том числе стандарты управления рисками ISO 31000, модель Cynefin, методологии Agile, Scrum, XP, CRISP-DM, CRISP-ML(Q), Lean DS).

## 5. Структура банка вопросов

Банк вопросов состоит из **двух групп**, разделённых по типу проверяемого результата. Внутри каждой группы вопросы разбиты на **5 тематических подгрупп**, распределённых по 2 модулям курса пропорционально их трудоёмкости:

| Тип вопросов | Назначение | Тип вопроса (Moodle) | Расположение |
|---|---|---|---|
| Теоретические | Проверка владения понятийным аппаратом («дайте определение понятия…») | essay (открытый ответ, оценивается вручную по эталонному определению) | `Question_bank_theory/kim_kolokvium_7-8_theory_gr1.xml` … `gr5.xml` |
| Проблемные (кейсы) | Проверка умения применять понятия и методы для анализа и решения проблемных ситуаций AI-проекта | essay (открытый развёрнутый ответ, оценивается вручную по критериям) | `Question_bank_cases/kim_kolokvium_7-8_case_gr1.xml` … `gr5.xml` |

Каждой из 5 подгрупп `gr{N}.xml` соответствует конкретная тема в рамках одного из модулей:

| № группы | Модуль | Тема группы |
|---:|---|---|
| gr1 | 7 | Управление рисками AI-проекта (виды, источники, реестр рисков) |
| gr2 | 7 | Взаимодействие с внешними участниками и управление стейкхолдерами |
| gr3 | 8 | Гибкие методологии управления проектами в ИИ-контексте (Agile, Scrum, XP, Lean DS) |
| gr4 | 8 | Выбор методологии для AI-проекта (модель Cynefin, сравнение подходов) |
| gr5 | 8 | Инструментальная среда проектного менеджмента и планирование спринта |

В теоретической группе каждый файл содержит 2–3 вопроса на определение ключевых понятий темы. В группе кейсов каждый файл содержит проблемную ситуацию, требующую развёрнутого ответа с обоснованием решения.

## 6. Формирование варианта коллоквиума

1. Из всех 5 подгрупп (охватывающих оба модуля) выбирается случайным образом:
   - один теоретический вопрос из каждой подгруппы (итого 5 вопросов);
   - одна проблемная ситуация из каждой подгруппы (итого 5 кейсов).
2. Типовой вариант коллоквиума включает 5 теоретических вопросов и 5 кейсов, охватывающих оба модуля курса.
3. При проведении в формате Moodle-квиза случайная выборка настраивается через категории вопросов, соответствующие файлам `gr{N}.xml`; из каждой выбранной категории берётся один случайный вопрос.
4. При проведении в устном формате вопросы того же банка используются для формирования билетов (см. **[kim_8_Colloq_var.md](kim_8_Colloq_var.md)**, вариант «Устное собеседование»).

## 7. Требования к процедуре проведения

### Условия проведения (формат Moodle)

- коллоквиум проводится в системе Moodle (или аналогичной LMS) с использованием банка вопросов типа «эссе»;
- ответы на вопросы проверяются преподавателем вручную по эталонным определениям (теоретический блок) и по критериям оценивания (блок кейсов), указанным в поле `graderinfo` каждого вопроса;
- рекомендуемое время выполнения — 3–4 минуты на теоретический вопрос, 8–10 минут на кейс.

### Условия проведения (устный формат)

- обучающийся получает билет из банка вопросов (см. раздел 6) и время на подготовку (10–15 минут);
- устный ответ включает раскрытие определений и разбор решения проблемных ситуаций;
- преподаватель вправе задавать уточняющие вопросы по любому из двух модулей курса.

## 8. Требования к ответу

Ответ на теоретический вопрос должен:

1. раскрывать смысл понятия своими словами (дословное совпадение с эталонной формулировкой не требуется);
2. быть точным и не содержать подмены понятия смежным термином.

Ответ на кейс должен:

1. демонстрировать понимание контекста проблемной ситуации;
2. содержать системный и полный анализ, а не единичное поверхностное решение;
3. предлагать обоснованное и реалистичное решение;
4. при необходимости — сопровождаться расчётами (например, для кейсов по оценке рисков);
5. учитывать специфику AI-проектов (данные, модели, итеративность, MLOps).

## 9. Процедура оценивания

Оценивание описано в `kim_8_Colloq_mark.md`. Итоговый балл коллоквиума складывается из суммы баллов за 5 теоретических вопросов (максимум 2 балла за вопрос) и 5 кейсов (максимум 5 баллов за кейс) с последующим приведением к единой 10-балльной шкале ФОС.
