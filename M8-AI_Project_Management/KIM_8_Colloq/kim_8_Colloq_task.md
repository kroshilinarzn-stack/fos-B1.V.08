# КИМ 8. Коллоквиум по дисциплине «Управление проектами в сфере ИИ» (модуль 8)

## 1. Паспорт КИМ

| Параметр | Значение |
|---|---|
| Дисциплина | Управление проектами в сфере ИИ |
| Модуль | Модуль 8. Управление проектами в сфере искусственного интеллекта: методы, подходы, инструментарий |
| Вид оценочного средства | Коллоквиум (опрос по банку теоретических и проблемных вопросов) |
| Вид контроля | Рубежный контроль |
| Рекомендуемый формат выполнения | Индивидуально |
| Рекомендуемая среда | LMS Moodle (банк вопросов типа «эссе») либо устный опрос с использованием того же банка вопросов |
| Основной артефакт | Письменные (в Moodle) или устные ответы на вопросы двух типов: теоретические и проблемные (кейсы) |
| Рекомендуемая трудоёмкость | 1,5 академических часа |

## 2. Назначение КИМ

КИМ предназначен для рубежной проверки усвоения материала модуля 8: выбора методологии управления AI-проектом (в том числе на основе модели Cynefin), гибких методологий (Agile, Scrum, XP, Lean DS), инструментальной среды проектного менеджмента, планирования спринтов и Scrum-церемоний, а также принятия управленческих решений о методологии в комплексных практических ситуациях. Коллоквиум проверяет как владение терминологией (теоретический блок), так и умение обоснованно выбирать и применять методологию управления для конкретного AI-проекта (блок кейсов).

## 4. Источники информации

Для подготовки к коллоквиуму обучающиеся используют материалы лекций, самостоятельной работы, лабораторной и практической работ модуля 8, а также источники, указанные в паспортах модульных КИМ дисциплины «Управление проектами в сфере ИИ».

## 5. Структура банка вопросов

Банк вопросов состоит из **двух групп**, разделённых по типу проверяемого результата. Внутри каждой группы вопросы разбиты на **15 тематических подгрупп**, соответствующих содержанию модуля 8:

| Тип вопросов | Назначение | Тип вопроса (Moodle) | Расположение |
|---|---|---|---|
| Теоретические | Проверка владения понятийным аппаратом («дайте определение понятия…») | essay (открытый ответ, оценивается вручную по эталонному определению) | `Question_bank_theory/kim_8_Colloq_theory_gr1.xml` … `gr15.xml` |
| Проблемные (кейсы) | Проверка умения обоснованно выбирать и применять методологию управления AI-проектом | essay (открытый развёрнутый ответ, оценивается вручную по критериям) | `Question_bank_cases/kim_8_Colloq_case_gr1.xml` … `gr15.xml` |

| № группы | Тема группы |
|---:|---|
| gr1 | Модель Cynefin и выбор подхода |
| gr2 | Гибкие методы: Agile |
| gr3 | Практики Scrum |
| gr4 | XP (Extreme Programming) |
| gr5 | Lean DS — адаптация Lean и Agile к ИИ |
| gr6 | Инструментальная среда управления проектами |
| gr7 | КСУП-системы |
| gr8 | Сравнительный анализ методологий для AI-проекта |
| gr9 | Выбор методологии под ограничения ресурсов |
| gr10 | Планирование спринта |
| gr11 | Ежедневные стендапы |

| gr13 | Комплексный кейс: выбор подхода |

В теоретической группе каждый файл содержит 1–2 вопроса на определение ключевых понятий темы. В группе кейсов каждый файл содержит проблемную ситуацию, требующую развёрнутого ответа с обоснованием решения.

## 6. Формирование варианта коллоквиума

1. Из случайно выбранных подгрупп (рекомендуется не менее 6 из 15) выбирается один теоретический вопрос и одна проблемная ситуация.
2. Типовой вариант коллоквиума включает 6–8 теоретических вопросов и 6–8 кейсов, охватывающих основное содержание модуля 8.
3. При проведении в формате Moodle-квиза случайная выборка настраивается через категории вопросов, соответствующие файлам `gr{N}.xml`.
4. При проведении в устном формате вопросы того же банка используются для формирования билетов (см. `kim_8_Colloq_var.md`, вариант «Устное собеседование»).

## 7. Требования к процедуре проведения

### Условия проведения (формат Moodle)

- коллоквиум проводится в системе Moodle (или аналогичной LMS) с использованием банка вопросов типа «эссе»;
- ответы на вопросы проверяются преподавателем вручную по эталонным определениям (теоретический блок) и по критериям оценивания (блок кейсов), указанным в поле `graderinfo` каждого вопроса;
- рекомендуемое время выполнения — 3–4 минуты на теоретический вопрос, 8–10 минут на кейс.

### Условия проведения (устный формат)

- обучающийся получает билет из банка вопросов (см. раздел 6) и время на подготовку (15 минут);
- устный ответ включает раскрытие определений и обоснованный выбор методологии для проблемной ситуации;
- преподаватель вправе задавать уточняющие вопросы по любой из пяти тем модуля, в том числе на связь с ранее изученными модулями курса (например, с модулем 6 или модулем 7).

## 8. Требования к ответу

Ответ на теоретический вопрос должен:

1. раскрывать смысл понятия своими словами (дословное совпадение с эталонной формулировкой не требуется);
2. быть точным и не содержать подмены понятия смежным (например, Agile и Scrum, спринт и итерация без уточнения различий).

Ответ на кейс должен:

1. демонстрировать понимание контекста проблемной ситуации;
2. содержать сравнительный анализ применимых методологий, а не выбор «по умолчанию»;
3. предлагать обоснованное решение с учётом ограничений задачи (сроки, регуляторные требования, неопределённость).

## 9. Процедура оценивания

Оценивание описано в `kim_8_Colloq_mark.md`. Итоговый балл коллоквиума складывается из суммы баллов за теоретические вопросы (максимум 2 балла за вопрос) и кейсы (максимум 5 баллов за кейс) выбранного варианта с последующим приведением к единой 10-балльной шкале ФОС.

## Пересчёт в БРС

КИМ оценивается по единой 10-балльной шкале ФОС. Вес данного КИМ в итоговой балльно-рейтинговой системе определяется в документе `docs/evaluation_policy.md`.

```text
балл_БРС = (балл_КИМ / 10) × вес_КИМ
```

Так как контроль по модулю является рубежным, дополнительно допускается собственный алгоритм формирования рубежной оценки (например, среднее по нескольким КИМ блока), который фиксируется в `docs/evaluation_policy.md`.
